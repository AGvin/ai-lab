# Параметроефективний fine-tuning

Parameter-Efficient Fine-Tuning, або PEFT, адаптує модель, навчаючи невелику частину параметрів або додаткові модулі та зберігаючи більшість базових ваг замороженими.

## Переклади

- [English](../../)
- Українська

## Основна ідея

Повний fine-tuning зберігає gradients і стан optimizer для всієї моделі. PEFT зменшує використання пам’яті під час навчання та створює менші артефакти адаптації. Методи включають LoRA, adapters, prefix tuning, prompt tuning і вибіркове оновлення параметрів.

## Практичне застосування

- Адаптувати великі моделі на обмеженому обладнанні.
- Підтримувати кілька спеціалізованих adapters для однієї базової моделі.
- Поширювати невеликі файли адаптації замість повних checkpoints.
- Експериментувати зі змінами домену чи стилю з меншими витратами.

## Компроміси й обмеження

Для багатьох задач PEFT може наближатися за якістю до повного fine-tuning, але не є універсально рівноцінним. Adapter залежить від конкретної базової моделі й часто від конкретної архітектури або схеми назв модулів. Обслуговування кількох adapters може додавати складності runtime.

## Типові помилки

- Завантаження adapter для неправильної ревізії базової моделі.
- Припущення, що мала кількість навчуваних параметрів запобігає overfitting.
- Порівняння методів без однакових даних і оцінювання.
- Ігнорування того, що важливі ліцензії і adapter, і базової моделі.
