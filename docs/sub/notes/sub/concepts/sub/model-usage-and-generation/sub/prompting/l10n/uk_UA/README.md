# Промптинг

Промптинг — це надання інструкцій, контексту, прикладів і обмежень, які спрямовують модель до корисного результату.

## Переклади

- [English](../../)
- Українська

## Основна ідея

Сильний промпт визначає задачу досить чітко, щоб моделі не доводилося вгадувати істотні вимоги. Він відокремлює інструкції від вихідного матеріалу, задає бажаний вихід і надає лише потрібний контекст. Якість промпту важлива, але не компенсує відсутніх знань, недостатніх можливостей моделі, недоступних інструментів або ненадійних вихідних даних.

## Корисні складники промпту

- **Ціль:** що потрібно виконати.
- **Контекст:** факти й вихідний матеріал для задачі.
- **Обмеження:** межі, винятки, правила безпеки й критерії прийняття.
- **Контракт виходу:** формат, довжина, мова, схема або аудиторія.
- **Приклади:** демонстрації, коли бажаний шаблон складно описати.

## Практичне застосування

- Формулюйте найважливішу вимогу прямо.
- Відмежовуйте недовірені документи чи цитований вміст, щоб їх не сплутали з інструкціями.
- Просіть явно вказувати невизначеність за неповних доказів.
- Діліть складну роботу на перевірні етапи або використовуйте агентний робочий процес.
- Оцінюйте промпти на сталому наборі репрезентативних випадків до використання як виробничої конфігурації.

## Компроміси й обмеження

Дуже директивні промпти підвищують послідовність, але можуть знижувати гнучкість. Довгі промпти займають контекст і можуть містити суперечливі інструкції. Поведінка також змінюється між сімействами й версіями моделей, тому промпти потребують регресійного тестування.

## Типові помилки

- Змішувати інструкції задачі з недовіреним вихідним текстом.
- Вимагати нечітких якостей на кшталт «краще» без критеріїв.
- Додавати багато надлишкових правил, що суперечать одне одному.
- Очікувати, що сам промптинг надасть актуальні або приватні знання.
