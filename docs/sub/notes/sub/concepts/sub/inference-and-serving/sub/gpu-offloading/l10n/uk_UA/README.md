# Вивантаження на GPU

Вивантаження на GPU розміщує частину або всі обчислення й ваги моделі на GPU, тоді як решта роботи залишається на CPU чи іншому пристрої.

## Переклади

- [English](../../)
- Українська

## Основна ідея

Коли модель не вміщується повністю у VRAM, середовище може розмістити вибрані шари на GPU, а решту залишити в системній пам’яті. Більша кількість шарів на GPU зазвичай підвищує швидкість, але передавання даних і синхронізація можуть зменшити користь від невдало вибраного поділу.

## Практичне застосування

- Запускати моделі, більші за доступну VRAM.
- Балансувати пам’ять ваг моделі, KV-кешу й пакетів.
- Ефективно використовувати інтегровані GPU або системи зі спільною пам’яттю.
- Порівнювати повну, часткову й відсутню роботу на GPU.

## Міркування щодо проєктування

Залишайте VRAM для контексту, буферів середовища й одночасних запитів. Відстежуйте фактичне виділення пам’яті, а не довіряйте лише налаштованій кількості шарів. На швидкість часткового вивантаження впливають пропускна здатність PCIe та пам’яті й продуктивність CPU.

## Компроміси й обмеження

Часткове вивантаження може бути значно повільнішим за повне розміщення на GPU. Деякі шари чи операції неможливо ефективно розділити. Системи зі спільною пам’яттю усувають явні межі передавання, але зберігають обмеження пропускної здатності й місткості.

## Типові помилки

- Заповнювати VRAM вагами, не залишаючи місця для KV-кешу.
- Припускати, що більша кількість вивантажених шарів завжди підвищує продуктивність.
- Порівнювати системи з різними розмірами контексту й пакета.
- Ігнорувати теплове дроселювання й сталі обмеження потужності.
