# Завантаження моделі

Завантаження моделі переміщує її артефакти зі сховища до RAM, VRAM, пам’яті прискорювача або адресного простору з відображенням пам’яті, щоб розпочати інференс.

## Переклади

- [English](../../)
- Українська

## Основна ідея

Завантаження охоплює читання файлів, виділення пам’яті, ініціалізацію структур середовища, передавання шарів на пристрої, а іноді й компіляцію ядер. Потрібна пам’ять перевищує розмір файла моделі, бо середовищу також потрібні буфери, кеші, метадані й тимчасові робочі області.

## Практичні міркування

- Швидкість сховища впливає на час холодного запуску.
- Відображення пам’яті може уникнути копіювання всієї моделі в анонімну RAM.
- Час передавання на GPU зростає з розміром моделі й обмеженням шини.
- Кілька процесів можуть спільно використовувати відображені ваги або дублювати їх.
- Деякі середовища компілюють оптимізовані графи під час першого використання.

## Практичне застосування

Окремо вимірюйте холодний і теплий запуск та затримку першого запиту. Залишайте резерв пам’яті для ОС, кешу контексту й одночасних запитів. Для контейнерів перевіряйте межі пам’яті й розташування постійних кешів.

## Компроміси й обмеження

Постійно завантажена модель зменшує затримку, але безперервно займає пам’ять. Завантаження на вимогу заощаджує ресурси ціною довгого запуску. Відображення пам’яті залежить від кешування ОС і під навантаженням усе одно може спричиняти помилки сторінок.

## Типові помилки

- Оцінювати пам’ять лише за розміром файла.
- Запускати кілька процесів моделі, що дублюють ваги.
- Ігнорувати тимчасове сховище для перетворення чи компіляції.
- Тестувати лише повністю прогріту модель.
