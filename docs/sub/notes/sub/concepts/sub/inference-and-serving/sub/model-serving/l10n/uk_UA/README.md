# Обслуговування моделей

Обслуговування моделей надає доступ до інференсу через керований процес, API, чергу або середовище, що опрацьовує запити, ресурси, планування й результати.

## Переклади

- [English](../../)
- Українська

## Основна ідея

Рівень обслуговування перетворює завантажену модель на операційний сервіс. Він керує розбиранням запитів, пакетуванням, паралельністю, автентифікацією, тайм-аутами, спостережуваністю, версіями моделей, а іноді й маршрутизацією між репліками чи прискорювачами.

## Практичні обов’язки

- Завантажувати й утримувати моделі в пам’яті.
- Перевіряти розмір входу й підтримувані параметри.
- Планувати й пакетувати запити.
- Передавати токени потоком або повертати завершені виходи.
- Забезпечувати квоти й обмеження частоти.
- Записувати затримку, використання й збої.
- Безпечно розгортати нові версії моделей.

## Компроміси й обмеження

Локальне настільне середовище просте, але може не ізолювати кількох користувачів. Виробничі сервери поліпшують використання ресурсів і спостережуваність, проте ускладнюють розгортання й керування місткістю. Репліки моделей можуть дублювати великі обсяги пам’яті прискорювача.

## Типові помилки

- Відкривати сервер розроблення недовіреним мережам.
- Дозволяти необмежені довжини контексту або виходу.
- Розгортати нову версію моделі без регресійних тестів.
- Вимірювати швидкість моделі без затримки черги й мережі.
