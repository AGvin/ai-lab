# Пам’ять агента

Пам’ять агента зберігає інформацію поза межами поточного запиту, щоб вона могла впливати на подальші кроки або майбутні сесії.

## Переклади

- [English](../../)
- Українська

## Типи пам’яті

- **Робоча пам’ять:** тимчасова інформація, потрібна для поточної задачі.
- **Епізодична пам’ять:** записи про попередні взаємодії або виконані задачі.
- **Семантична пам’ять:** стійкі факти, уподобання або засвоєні знання предметної області.
- **Процедурна пам’ять:** придатні до повторного використання інструкції, робочі процеси або навички.

## Основна ідея

Пам’ять зазвичай реалізують через бази даних, системи пошуку, резюме або структуровані профілі, а не шляхом зміни ваг моделі. Система пам’яті має вирішувати, що зберігати, коли отримувати збережене, як його оновлювати й коли забувати.

## Практичне застосування

- Зберігати вподобання користувача за його явною згодою.
- Повторно використовувати перевірені висновки в довготривалому проєкті.
- Відновлювати роботу після скорочення контексту або перезапуску процесу.
- Отримувати попередні рішення, дотичні до поточної задачі.

## Компроміси й обмеження

Помилкова пам’ять може знову й знову спотворювати майбутні рішення. Тривале зберігання підвищує ризики для приватності та безпеки. Пошук додає затримку й може повертати недоречні або застарілі факти.

## Типові помилки

- Постійно зберігати кожне повідомлення розмови.
- Вважати згенероване резюме авторитетним фактом.
- Не записувати походження, часові позначки або рівень упевненості.
- Отримувати спогади, не перевіряючи, чи вони досі застосовні.
