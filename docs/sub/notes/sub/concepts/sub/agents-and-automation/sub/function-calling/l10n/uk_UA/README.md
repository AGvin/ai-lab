# Виклик функцій

Виклик функцій — це структурована форма виклику інструментів, за якої модель вибирає іменовану функцію й передає аргументи, що відповідають оголошеній схемі.

## Переклади

- [English](../../)
- Українська

## Основна ідея

Слово «функція» описує інтерфейс, доступний моделі; базова реалізація може викликати API, поставити завдання в чергу, виконати запит до бази даних або запустити локальний код. Основний застосунок і надалі відповідає за автентифікацію, авторизацію, перевірку, виконання й оброблення помилок.

## Практичне застосування

- Видобувати типізовані аргументи із запитів природною мовою.
- Спрямовувати запити до можливостей застосунку.
- Під’єднувати інтерфейси чату до бізнес-операцій.
- Створювати структуровані дані для етапу робочого процесу.
- Дозволяти моделі вибирати з невеликого набору схвалених дій.

## Рекомендації з проєктування

Використовуйте змістовні назви й точні описи параметрів. Віддавайте перевагу перелікам і типізованим полям замість рядків довільної форми. Робіть функції достатньо вузькими, щоб перевірки дозволів мали сенс. Повертайте компактні структуровані результати, які розрізняють успіх, невдачу й часткове завершення.

## Компроміси й обмеження

Коректний об’єкт виклику функції не означає, що намір безпечний або правильний. Схеми функцій можуть ставати великими, а численні подібні функції — знижувати точність вибору. Реалізації постачальників відрізняються суворістю дотримання схем.

## Типові помилки

- Використовувати модель як рівень авторизації.
- Передавати згенеровані шляхи до файлів, SQL або команди безпосередньо на виконання.
- Не повертати помилки інструментів до робочого процесу.
- Плутати запропонований виклик функції із завершеним виконанням.
