# Стан агента

Стан агента — це явні дані, які фіксують поточне положення робочого процесу, проміжні результати, рішення, очікувані дії та метадані виконання.

## Переклади

- [English](../../)
- Українська

## Основна ідея

Стан не повинен існувати лише всередині розмови з моделлю. Явний стан дає змогу відновлювати, перевіряти й тестувати агента, а також безпечніше повторювати його дії. Для коротких задач його можна зберігати в пам’яті, а для тривалої роботи — у базі даних, рушії робочих процесів або журналі подій.

## Типові поля стану

- Ідентифікатор задачі та поточний етап.
- Вхідні дані, перевірені вихідні дані й результати інструментів.
- Виконані та очікувані дії.
- Лічильники повторних спроб і відомості про помилки.
- Схвалені або відхилені людиною дії.
- Версії моделі, промпту й конфігурації.

## Практичне застосування

Використовуйте типізовані об’єкти стану й перевіряйте кожен перехід. Зберігайте посилання на великі артефакти замість їх повторного внесення до контексту моделі. Відокремлюйте авторитетний стан застосунку від згенерованих моделлю нотаток або резюме.

## Компроміси й обмеження

Збережений стан поліпшує відновлення та аудит, але створює питання еволюції схем, приватності, строків зберігання й конкурентного доступу. Стан може стати неузгодженим, якщо зовнішні побічні ефекти відбуваються без відповідних оновлень.

## Типові помилки

- Використовувати історію чату як єдине джерело істини.
- Зберігати в стані секрети або непотрібні персональні дані.
- Оновлювати стан до підтвердження успішності зовнішньої дії.
- Дозволяти кільком виконавцям змінювати одну задачу без координації.
