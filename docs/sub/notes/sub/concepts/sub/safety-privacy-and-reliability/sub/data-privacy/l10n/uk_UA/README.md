# Приватність даних

Приватність даних визначає збирання, оброблення, зберігання, передавання й видалення особистої, конфіденційної чи чутливої інформації.

## Переклади

- [English](../../)
- Українська

## Основна ідея

Робочий процес ШІ може розкривати дані через промпти, журнали постачальника, ембединги, кеші, трасування, навчальні набори або результати інструментів. Тому приватність потрібно проєктувати для всього шляху даних, а не лише кінцевої точки моделі.

## Практичні засоби контролю

- Мінімізуйте дані до надсилання моделі.
- За можливості вилучайте або токенізуйте ідентифікатори.
- Визначайте правила зберігання й видалення.
- Розділяйте орендарів і області дозволів.
- Документуйте політики використання даних постачальника.
- Зберігайте чутливі навантаження локально, коли цього потребують вимоги.

## Компроміси й обмеження

Вилучення деталей може зменшити корисність моделі. Локальне оброблення підвищує контроль, але додає операційної відповідальності. Анонімізація може не спрацювати, якщо решта ознак дає повторно ідентифікувати особу.

## Типові помилки

- Створювати ембединги конфіденційних документів без контролю доступу.
- Вважати псевдонімізовані дані анонімними.
- Безстроково зберігати промпти за замовчуванням.
- Надсилати повні записи, коли достатньо малої підмножини.
