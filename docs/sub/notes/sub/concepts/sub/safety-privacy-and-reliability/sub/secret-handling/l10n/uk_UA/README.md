# Робота із секретами

Робота із секретами охоплює зберігання, доставлення, використання, ротацію й редагування облікових даних, токенів, ключів та інших чутливих експлуатаційних значень.

## Переклади

- [English](../../)
- Українська

## Основна ідея

Моделі мають отримувати лише мінімальну похідну від секрету можливість, потрібну для задачі, а не необроблені довготривалі облікові дані. Рівень виконання може використати облікові дані, не розкриваючи їх у контексті prompt або output інструмента.

## Належна практика

- Зберігайте секрети у спеціалізованому secret manager.
- Використовуйте короткотривалі облікові дані з обмеженою областю дії.
- Редагуйте журнали, траси, помилки й результати інструментів.
- Не допускайте потрапляння секретів у prompts або embeddings.
- Ротуйте облікові дані після підозри на розкриття.
- Розділяйте production- і development-облікові дані.

## Компроміси й обмеження

Брокери секретів і короткотривалі токени додають інфраструктурної та інтеграційної роботи. Редагування має балансувати безпеку з достатньою для операторів діагностичною інформацією.

## Типові помилки

- Розміщення API keys у system prompts.
- Додавання файлів `.env` до контексту моделі.
- Запис повних HTTP headers або connection strings у журнали.
- Припущення, що приватний провайдер моделі робить розкриття секретів нешкідливим.
