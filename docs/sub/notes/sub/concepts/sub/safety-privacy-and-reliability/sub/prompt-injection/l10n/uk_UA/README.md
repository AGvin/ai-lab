# Prompt injection

Prompt injection — це input, призначений змусити систему ШІ ігнорувати передбачені інструкції, розкрити захищену інформацію або виконати неавторизовані дії.

## Переклади

- [English](../../)
- Українська

## Основна ідея

Модель обробляє інструкції та дані в одному загальному середовищі — токенах. Тому зловмисник може розмістити текст, схожий на інструкцію, у повідомленні користувача й спробувати перевизначити політику або переспрямувати використання інструментів. Ієрархія prompts допомагає, але не є повною межею безпеки.

## Поширені цілі

- Перевизначити правила системи або застосунку.
- Викрасти приховані prompts, секрети чи отримані дані.
- Спричинити небезпечні виклики інструментів.
- Обійти обмеження вмісту або бізнес-політики.
- Маніпулювати кінцевою відповіддю моделі.

## Зменшення ризику

Застосовуйте інструменти з найменшими привілеями, детерміновану авторизацію, надійні межі довіри, валідацію схем, фільтрацію output і схвалення людиною для дій із наслідками. Не розміщуйте секрети у видимому моделі контексті. Сприймайте output моделі як недовірений input для кожного інструмента.

## Компроміси й обмеження

Захист на основі prompt може послабити прості атаки, але не гарантує ізоляції. Агресивна фільтрація також може блокувати законні інструкції. Безпеку має забезпечувати навколишній застосунок.

## Типові помилки

- Прохання до моделі «ігнорувати шкідливі інструкції» як повне розв’язання проблеми.
- Надання необмежених shell-, database- або browser-інструментів.
- Повернення секретів у видимих моделі результатах інструментів.
- Сприйняття відмови в тестах як доказу безпеки.
