# LLM як суддя

LLM як суддя використовує мовну модель для оцінювання, ранжування, критики чи порівняння outputs іншої моделі або системи.

## Переклади

- [English](../../)
- Українська

## Основна ідея

Модель-суддя може оцінювати якості, які складно закодувати точними правилами: релевантність, повноту, стиль або відповідність rubric. Попарне порівняння часто стабільніше за присвоєння абсолютної числової оцінки.

## Належна практика

- Надавайте точну rubric і перелік свідчень, потрібних для оцінювання.
- Випадково змінюйте порядок відповідей, щоб виявляти позиційне упередження.
- За можливості приховуйте ідентичність моделі.
- Калібруйте за прикладами, перевіреними людьми.
- Для оцінювання з великим впливом використовуйте кількох суддів або повторні запуски.
- Зберігайте детерміновані перевірки фактів і схем.

## Компроміси й обмеження

Моделі-судді мають власні упередження, переваги й сліпі зони. Вони можуть віддавати перевагу багатослівним відповідям, знайомим стилям або outputs споріднених сімейств моделей. На суддю також може вплинути prompt injection усередині оцінюваної відповіді.

## Типові помилки

- Використання однієї моделі для генерації та оцінювання без калібрування.
- Сприйняття оцінок судді як об’єктивної істини.
- Запит на перевірку фактів без надання джерел.
- Ігнорування неузгоджених результатів повторних оцінювань.
