# Оцінювання

Evals — повторювані тести, що визначають, чи відповідає модель або система ШІ встановленим вимогам якості, безпеки, надійності й експлуатації.

## Переклади

- [English](../../)
- Українська

## Основна ідея

Оцінювання починається з конкретної задачі та критеріїв успіху. Воно надає репрезентативні inputs, запускає фіксовану конфігурацію й оцінює outputs детермінованими перевірками, review людиною, model judges або їх поєднанням. Evals повинні вимірювати повну систему, коли prompts, retrieval, tools і post-processing суттєво впливають на результат.

## Практичне застосування

- Порівнювати кандидатні моделі для workflow.
- Виявляти regressions після змін prompt або моделі.
- Вимірювати надійність tool calling і structured output.
- Тестувати hallucination, safety і prompt injection.
- Установлювати release gates для production changes.

## Належний дизайн оцінювання

Використовуйте реалістичні приклади, додавайте складні й негативні cases, зберігайте held-out set і записуйте версії моделі та конфігурації. Повідомляйте як aggregate metrics, так і важливі категорії відмов.

## Компроміси й обмеження

Evals наближено відтворюють реальне використання й можуть застарівати. Модель може overfit публічні benchmarks або багаторазово використаний внутрішній набір. Автоматичні scores можуть пропускати тонкі фактичні, стилістичні або safety problems.

## Типові помилки

- Тестувати лише прості happy-path prompts.
- Змінювати dataset під час порівняння версій моделей.
- Використовувати один середній score без аналізу відмов.
- Оцінювати модель, але не retrieval і tool execution.
