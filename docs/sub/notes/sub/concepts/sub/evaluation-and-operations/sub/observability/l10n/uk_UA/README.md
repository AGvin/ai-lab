# Спостережуваність

Спостережуваність — це здатність розуміти внутрішню поведінку системи ШІ за журналами, метриками, трасами, подіями та записаним станом.

## Переклади

- [English](../../)
- Українська

## Основна ідея

Збої ШІ часто розподілені між prompts, retrieval, викликами моделей, інструментами, валідацією та зовнішніми сервісами. Спостережуваність поєднує ці етапи, щоб оператори могли визначити, що сталося, чому це сталося і який компонент потребує виправлення.

## Корисні сигнали

- Обсяг запитів, затримка, вартість і використання токенів.
- Версії моделі та prompt.
- Retrieval-запити, кількість кандидатів та ідентифікатори джерел.
- Виклики інструментів, аргументи, результати й помилки.
- Результати валідації та кількість повторних спроб.
- Відгуки користувачів і метрики якості.

## Практичне застосування

Призначайте кожному виконанню workflow кореляційний ідентифікатор. Записуйте структуровані події, а не покладайтеся лише на журнали довільної форми. Створюйте панелі стану експлуатації та сповіщайте про незвичні патерни помилок, вартості, затримки чи відмов.

## Компроміси й обмеження

Докладна телеметрія може бути дорогою та розкривати чутливі prompts або дані. Потрібні редагування чутливого вмісту, вибірка, правила зберігання й контроль доступу. Спостережуваність пояснює записану поведінку, але не може відтворити дані, які не були зафіксовані.

## Типові помилки

- Запис повних секретів і персональних даних.
- Запис output моделі без версії моделі чи prompt.
- Моніторинг доступності API без моніторингу якості відповідей.
- Агрегування метрик без збереження трас окремих запитів для діагностики.
