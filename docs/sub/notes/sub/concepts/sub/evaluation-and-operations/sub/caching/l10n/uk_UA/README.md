# Кешування

Кешування повторно використовує раніше обчислені результати або оброблені дані, щоб скоротити затримку, вартість і повторну роботу.

## Переклади

- [English](../../)
- Українська

## Поширені об’єкти кешування

- Повні відповіді моделі.
- Префікс prompt або стан контексту.
- Embeddings і фрагменти документів.
- Результати retrieval.
- Відповіді інструментів та API.
- Артефакти завантаження моделі або скомпільовані kernels.

## Основна ідея

Ключ кешу має містити кожний input і деталь конфігурації, що змінюють правильний результат. Для систем ШІ це може включати версію моделі, system prompt, дозволи користувача, версію retrieval-індексу, temperature, локаль і стан інструментів.

## Практичне застосування

Кешуйте детерміновані або маловаріативні запити зі стабільними даними. Визначте правила завершення строку дії та інвалідації до розгортання. Ізолюйте дані користувачів і tenants та не кешуйте чутливий вміст, якщо ризик перевищує користь.

## Компроміси й обмеження

Кешування може повертати застарілі або неавторизовані результати. Великі ключі й низька частка влучень додають складності без користі. Кешування генеративних outputs також зменшує варіативність, що може бути як бажаним, так і небажаним.

## Типові помилки

- Пропуск ідентичності чи дозволів користувача в ключі.
- Повторне використання результатів після змін prompt або моделі.
- Безстрокове кешування відповідей із помилками.
- Сприйняття інвалідації кешу як другорядної задачі.
