# Оцінювання retrieval

Оцінювання retrieval визначає, чи повертає пошукова або RAG-система релевантні, достатні, правильно ранжовані й авторизовані свідчення.

## Переклади

- [English](../../)
- Українська

## Основні метрики

- **Recall at k:** чи з’являються релевантні свідчення серед перших `k` результатів.
- **Precision at k:** скільки отриманих елементів є релевантними.
- **Mean reciprocal rank:** наскільки рано з’являється перший релевантний результат.
- **NDCG:** якість ранжування, коли релевантність має кілька рівнів.
- **Покриття контексту:** чи достатньо отриманих свідчень для відповіді на запитання.

## Основна ідея

Якість кінцевої відповіді може приховувати збої retrieval, адже модель здатна відповісти з попередніх знань або згалюцинувати. Retrieval і генерацію слід оцінювати окремо, а потім разом.

## Практичне застосування

Створюйте мітки релевантності «запит — документ» із реальних задач. Додавайте точні ідентифікатори, перефразування, багатодокументні запитання, випадки без відповіді та перевірки контролю доступу. Порівнюйте лексичні, семантичні, гібридні й reranked-системи.

## Компроміси й обмеження

Мітки релевантності можуть бути суб’єктивними, а один запит може мати багато прийнятних наборів свідчень. Offline-метрики можуть не відображати, як генератор використовує контекст.

## Типові помилки

- Позначення лише одного уривка релевантним, коли відповідь підтверджують кілька.
- Вимірювання top-one accuracy для багатодокументних запитань.
- Ігнорування неавторизованих, але дуже релевантних результатів.
- Налаштування retrieval на кінцевому тестовому наборі.
