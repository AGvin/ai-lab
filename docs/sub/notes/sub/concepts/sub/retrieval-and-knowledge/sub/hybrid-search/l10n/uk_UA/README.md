# Гібридний пошук

Гібридний пошук поєднує семантичний векторний retrieval із лексичним, щоб результати враховували і значення, і точні збіги термінів.

## Переклади

- [English](../../)
- Українська

## Основна ідея

Два retrievers створюють списки кандидатів або оцінки, які потрібно об’єднати. Поширені методи включають зважене злиття оцінок, reciprocal rank fusion або отримання окремих наборів кандидатів із передаванням до reranker. Найкращий метод залежить від важливості точних ідентифікаторів, значення природної мови або обох чинників.

## Практичне застосування

- Пошук корпоративних документів із технічними термінами.
- Retrieval коду й журналів.
- Каталоги продуктів з описами та SKU.
- RAG-системи, що мають обробляти і запитання, і точні посилання.

## Компроміси й обмеження

Гібридний retrieval додає складності індексування, налаштування й оцінювання. Необроблені векторні та лексичні оцінки зазвичай мають різні шкали, тому їх не слід додавати без нормалізації чи rank-based fusion. Невдале злиття може послабити переваги обох систем.

## Належна практика

Окремо оцінюйте baseline лише з лексичним, лише із семантичним і з гібридним пошуком. Перевіряйте, які типи запитів покращилися чи погіршилися. Узгоджено застосовуйте metadata й permission filters до обох шляхів retrieval.

## Типові помилки

- Довільна вага 50/50 без тестування.
- Повернення дубльованих chunks з обох retrievers.
- Оцінювання лише якості кінцевої відповіді з приховуванням збоїв retrieval.
- Різні правила контролю доступу для кожного індексу.
