# Керування вартістю

Керування вартістю вимірює, розподіляє, прогнозує та контролює витрати на моделі, інфраструктуру, сховище, retrieval та інструменти.

## Переклади

- [English](../../)
- Українська

## Джерела витрат

- Вхідні, вихідні, кешовані та reasoning-токени.
- Час роботи GPU або CPU й невикористана потужність.
- Операції embedding і reranking.
- Векторне сховище, журнали й траси.
- Зовнішні API та виклики інструментів.
- Перевірка людиною й експлуатаційне обслуговування.

## Основна ідея

Найдешевший запит до моделі не обов’язково створює найдешевший успішний workflow. Слабка модель може потребувати більше повторних спроб, виправлень людиною чи більших prompts. Тому вартість слід вимірювати на один прийнятий результат задачі.

## Практичне застосування

Позначайте використання за користувачем, проєктом, моделлю та workflow. Установлюйте бюджети й сповіщення. Кешуйте стабільну роботу, спрямовуйте прості задачі до менших моделей, обмежуйте контекст і відстежуйте аномальні цикли агентів.

## Компроміси й обмеження

Агресивне скорочення витрат може знизити якість, надійність або приватність. Локальна інфраструктура замінює плату за API витратами на обладнання, електроенергію, обслуговування та планування потужностей.

## Типові помилки

- Підрахунок токенів моделі з ігноруванням вартості інструментів і людської праці.
- Порівняння локального й хостингового inference лише за граничною ціною запиту.
- Оптимізація середньої вартості, коли рідкісні помилки надзвичайно дорогі.
- Необмежені output і повторні спроби.
