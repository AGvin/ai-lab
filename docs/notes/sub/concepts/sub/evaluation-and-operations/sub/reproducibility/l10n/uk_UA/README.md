# Відтворюваність

Відтворюваність — це здатність повторити експеримент або workflow за задокументованих умов і отримати зіставні результати.

## Переклади

- [English](../../)
- Українська

## Основна ідея

Генеративні системи ймовірнісні, а провайдери можуть оновлювати моделі, тому побайтово однаковий output не завжди можливий. Практична відтворюваність означає запис достатньої інформації для пояснення й порівняння змін: версії моделі, prompt, параметрів, даних, інструментів, runtime, обладнання та методу оцінювання.

## Практичні вимоги

- За можливості фіксуйте версії моделі й залежностей.
- Зберігайте prompts і схеми в системі контролю версій.
- Версіонуйте datasets і retrieval-індекси.
- Записуйте seeds і параметри sampling.
- Фіксуйте конфігурацію обладнання та runtime для тестів продуктивності.
- Зберігайте необроблені результати до агрегування.

## Компроміси й обмеження

Хостингові моделі можуть змінюватися за стабільними назвами. Паралельні апаратні kernels можуть бути недетермінованими. Точна відтворюваність може бути дорогою або неможливою, тому визначте прийнятний допуск для задачі.

## Типові помилки

- Запис лише назви сімейства моделі.
- Одночасна зміна prompts і datasets у порівнянні.
- Припущення, що seed гарантує однаковий хостинговий output.
- Публікація агрегованих оцінок без необробленої конфігурації.
