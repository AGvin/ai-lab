# Трасування

Трасування записує впорядковану послідовність викликів моделей, retrieval-операцій, виконань інструментів, валідацій і переходів стану для одного виконання workflow.

## Переклади

- [English](../../)
- Українська

## Основна ідея

Траса показує причинну структуру, якої не видно з агрегованих метрик. Вона може виявити, що погана відповідь почалася зі слабкого retrieval-запиту, timeout інструмента, неправильно сформованого аргументу чи рішення моделі на основі застарілого стану.

## Типові дані траси

- Час початку й завершення кожного span.
- Батьківсько-дочірні зв’язки між операціями.
- Версії моделі, prompt, інструмента й конфігурації.
- Метадані токенів, вартості, затримки та статусу.
- Ідентифікатори джерел і результати валідації.
- Відредаговані помилки та історія повторних спроб.

## Практичне застосування

Використовуйте траси для налагодження циклів агентів, порівняння версій моделей, аналізу затримки та створення evaluation datasets із реальних помилок. Зберігайте достатньо контексту для відтворення проблеми без зайвого чутливого вмісту.

## Компроміси й обмеження

Висококардинальні дані трас дорогі. Повне збереження prompt покращує діагностику, але підвищує ризик для приватності. Вибірка може пропускати рідкісні збої, тому критичні помилки слід зберігати окремо.

## Типові помилки

- Запис викликів інструментів без результатів або статусу.
- Пропуск батьківсько-дочірніх зв’язків.
- Збереження необроблених облікових даних в атрибутах трас.
- Використання трас як постійної пам’яті агента.
