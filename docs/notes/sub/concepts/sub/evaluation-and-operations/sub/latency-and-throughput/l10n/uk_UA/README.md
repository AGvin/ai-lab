# Затримка та пропускна здатність

Затримка та пропускна здатність описують два пов’язані, але конкуруючі виміри продуктивності системи: скільки триває один запит і скільки роботи система загалом виконує за певний час.

## Переклади

- [English](../../)
- Українська

## Основна ідея

Інтерактивні застосунки зазвичай надають пріоритет часу до першого токена й плавності відповіді. Пакетні системи — загальній кількості завершених токенів або запитів. Збільшення розміру batch і глибини черги може підвищити пропускну здатність, водночас змушуючи окремих користувачів чекати довше.

## Практичне застосування

- Визначайте цілі рівня сервісу для затримки p50, p95 і p99.
- Вимірюйте пропускну здатність за потрібної цільової затримки.
- Тестуйте реалістичні довжини prompt і output.
- Розділяйте час черги, retrieval, моделі, інструментів і мережі.
- Оцінюйте тривале навантаження, а не короткі сплески.

## Компроміси й обмеження

Використання обладнання, паралельність, batching, довжина контексту й довжина output впливають на це співвідношення. Одне значення «токенів за секунду» не може відобразити якість багатокористувацького сервісу.

## Типові помилки

- Максимізація пропускної здатності без обмеження затримки.
- Порівняння систем із різними навантаженнями.
- Публікація середніх значень без поведінки хвостів розподілу.
- Ігнорування невдалих запитів і timeout.
