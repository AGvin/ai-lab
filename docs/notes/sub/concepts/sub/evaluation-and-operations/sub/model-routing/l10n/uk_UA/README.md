# Маршрутизація моделей

Маршрутизація моделей динамічно вибирає модель або конфігурацію для кожного запиту відповідно до типу задачі, ризику, вартості, затримки чи вимог до можливостей.

## Переклади

- [English](../../)
- Українська

## Основна ідея

Маршрутизатор може застосовувати явні правила, вибір користувача, класифікатори, оцінки впевненості або поетапне підвищення рівня. Прості задачі спрямовуються до швидкої недорогої моделі, а складні чи ризиковані — до потужнішої моделі або на перевірку людиною.

## Практичні шаблони

- Спрямовувати зображення лише до мультимодальних моделей.
- Передавати extraction невеликій моделі зі structured output.
- Підвищувати рівень для невдалих задач або задач із низькою впевненістю.
- Надавати перевагу локальним моделям для чутливих даних.
- Використовувати резервного провайдера під час збоїв.

## Компроміси й обмеження

Маршрутизація додає помилки класифікації та експлуатаційну складність. Дешева модель може помилково визнати задачу простою. Різні моделі можуть створювати неузгоджений стиль, схеми або безпекову поведінку.

## Належна практика

Записуйте рішення маршрутизації та результати. Оцінюйте повну політику відносно baseline з однією моделлю. Визначте жорсткі обмеження можливостей до оптимізації вартості й не допускайте перетину заборонених меж чутливими запитами.

## Типові помилки

- Маршрутизація лише за довжиною prompt.
- Дозвіл недовіреному prompt вибирати привілейовану модель або набір інструментів.
- Вимірювання зниження вартості без урахування регресій якості.
- Відсутність обробки версій і доступності.
