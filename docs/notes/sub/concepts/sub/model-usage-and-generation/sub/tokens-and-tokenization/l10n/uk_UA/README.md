# Токени й токенізація

Токени — це одиниці, які модель читає й генерує. Токенізація перетворює текст, код, розділові знаки, а іноді й пробіли на ідентифікатори токенів із фіксованого словника.

## Переклади

- [English](../../)
- Українська

## Основна ідея

Токен не тотожний слову чи символу. Поширене слово може відповідати одному токену, тоді як рідкісні слова, ідентифікатори, URL або нелатинський текст можуть ділитися на кілька. Моделі працюють із послідовностями токенів, а не сирим текстом, тому токенізація безпосередньо впливає на використання контексту, вартість генерації, затримку й надійність роботи з незвичними рядками.

## Ключові положення

- Різні сімейства моделей можуть по-різному токенізувати той самий текст.
- API може окремо рахувати вхідні токени, згенеровані вихідні токени й приховані токени міркування.
- Код, JSON, довгі числа й повторювані пробіли можуть займати більше контексту, ніж підказує їхня візуальна довжина.
- Межі токенів можуть впливати на правопис, копіювання, арифметику й задачі з точними рядками.

## Практичне застосування

- Оцінювати, чи вмістяться промпт і очікувана відповідь у контекстне вікно моделі.
- Порівнювати вартість API за кількістю токенів, а не символів.
- Аналізувати неочікувано дорогі промпти, особливо журнали, код і великі документи.
- Проєктувати розміри фрагментів для пошукових систем за токенізатором генеративної моделі або моделі ембедингів.

## Компроміси й обмеження

Більше словники можуть ефективно подавати часті рядки, але потребують більших таблиць ембедингів. Менші словники ділять більше тексту на частини й можуть збільшувати довжину послідовності. Тому кількість токенів залежить від моделі, і її не слід оцінювати токенізатором іншої моделі.

## Типові помилки

- Вважати одне слово одним токеном.
- Припускати однакову кількість токенів у різних постачальників.
- Ігнорувати бюджети вихідних токенів і токенів міркування під час визначення розміру запиту.
