# Параметри семплювання

Параметри семплювання керують тим, як генеративна модель вибирає наступний токен зі свого розподілу ймовірностей.

## Переклади

- [English](../../)
- Українська

## Основні засоби керування

- **Температура:** перемасштабує ймовірності токенів. Нижчі значення зазвичай роблять вихід зосередженішим і повторюванішим, а вищі збільшують варіативність.
- **Top-p:** обмежує вибір найменшим набором токенів, сукупна ймовірність якого досягає порога.
- **Top-k:** обмежує вибір `k` токенами з найвищою ймовірністю.
- **Seed:** може зробити запуски відтворюванішими, якщо постачальник, модель і шлях виконання підтримують детерміноване початкове значення.
- **Штрафи за повторення або частоту:** перешкоджають повторенню токенів чи фраз.

## Практичне застосування

Використовуйте налаштування з малою варіативністю для видобування, змін коду, класифікації та структурованого виводу. Помірна варіативність доречна для мозкового штурму, творчого письма чи створення різноманітних кандидатів. Змінюйте по одному параметру й оцінюйте результати на сталому тестовому наборі.

## Компроміси й обмеження

Низька температура не робить відповідь правдивою, а висока не створює справжніх знань. Назви й поведінка параметрів різняться між середовищами. Навіть детермінований seed може давати інші результати після зміни моделі, обладнання чи постачальника.

## Типові помилки

- Поєднувати агресивні зміни температури, top-p і top-k без вимірювання.
- Використовувати високу випадковість для задач із точними схемами.
- Припускати, що нульова температура гарантує ідентичний вихід.
- Налаштовувати семплювання до виправлення нечітких інструкцій або відсутнього контексту.
