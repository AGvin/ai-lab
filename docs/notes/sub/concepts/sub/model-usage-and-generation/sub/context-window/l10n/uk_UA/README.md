# Контекстне вікно

Контекстне вікно — це максимальний обсяг токенізованої інформації, який модель може врахувати в одному запиті, включно з інструкціями, історією розмови, знайденими документами, результатами інструментів і згенерованим виходом.

## Переклади

- [English](../../)
- Українська

## Основна ідея

Більше заявлене контекстне вікно дає змогу передати більше матеріалу, але не гарантує однаково якісного використання кожної деталі. Доречні факти можуть загубитися серед стороннього вмісту, повторюваних інструкцій або інформації далеко від поточного запитання. Тому ефективний контекст є водночас обмеженням місткості й задачею відбору інформації.

## Що займає контекст

- Системні інструкції та інструкції розробника.
- Історія розмови користувача й асистента.
- Схеми й результати інструментів, файли, зображення та знайдені фрагменти.
- Заплановане або згенероване завершення моделі.

## Практичне застосування

- Залишайте місце для виходу замість заповнення всього вікна вхідними даними.
- Узагальнюйте або вилучайте застарілу історію розмови.
- Отримуйте лише докази, дотичні до поточної задачі.
- Чітко розміщуйте критичні обмеження й повторюйте їх лише за потреби.
- Тестуйте довгий контекст із реалістичними документами, а не покладайтеся лише на опубліковану межу.

## Компроміси й обмеження

Довший контекст зазвичай збільшує використання пам’яті, час оброблення й вартість. У деяких середовищах зі зростанням контексту збільшується KV-кеш. Поблизу практичних меж моделі також можуть гірше пригадувати інформацію або дотримуватися інструкцій.

## Типові помилки

- Вважати межу контексту надійною робочою місткістю.
- Додавати кожен доступний документ замість вибіркового пошуку.
- Забувати, що очікуваний вихід також витрачає токени.
- Припускати, що старі повідомлення чату залишаються доступними після скорочення чи узагальнення.
