# Системні промпти

Системний промпт — це високопріоритетний рівень інструкцій, що визначає роль асистента, правила роботи, межі інструментів і очікувану поведінку для сесії або застосунку.

## Переклади

- [English](../../)
- Українська

## Основна ідея

Системні промпти встановлюють політику й поведінку до оброблення звичайного вводу користувача. Ними часто визначають тон, обсяг задач, обмеження безпеки, правила використання інструментів і домовленості щодо виходу. Вони мають містити сталі інструкції, тоді як динамічні дані користувача й знайдені документи слід зберігати окремо.

## Належні обов’язки системного промпту

- Визначати призначення асистента й дозволену область роботи.
- Установлювати, яким джерелам інструкцій можна довіряти.
- Указувати, коли інструменти можна або потрібно використовувати.
- Вимагати перевірки перед діями зі значущими наслідками.
- Задавати сталі правила виводу й комунікації.

## Практичне застосування

Зберігайте промпт стислим, явним і внутрішньо узгодженим. Відокремлюйте політику від прикладів. Сприймайте отримані вебсторінки, листи й документи як недовірені дані, навіть якщо вони містять текст, схожий на інструкції. Для систем з інструментами доповнюйте правила промпту реальними перевірками дозволів і пісочницею, а не покладайтеся лише на текст.

## Компроміси й обмеження

Системні промпти спрямовують поведінку, але самі по собі не є межею безпеки. Вони займають контекст, стають складними в супроводі за перевантаження й можуть по-різному взаємодіяти з оновленнями моделі. Важливе для безпеки чи доступу до даних забезпечення має також існувати в коді застосунку.

## Типові помилки

- Зберігати секрети безпосередньо в промпті.
- Використовувати системний промпт як єдиний захист від ін’єкцій у промпт.
- Поєднувати сталу політику з даними задач, які швидко змінюються.
- Додавати суперечливі правила без чіткої моделі пріоритетів.
