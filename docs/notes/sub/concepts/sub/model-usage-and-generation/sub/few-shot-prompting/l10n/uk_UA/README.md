# Промптинг із кількома прикладами

Промптинг із кількома прикладами спрямовує модель за допомогою невеликої кількості пар вхідних і вихідних даних, що демонструють бажану поведінку.

## Переклади

- [English](../../)
- Українська

## Основна ідея

Приклади передають закономірності, які складно повністю описати прозою: межі класифікації, правила форматування, термінологію або стиль перетворення. Модель виводить шаблон задачі з демонстрацій і застосовує його до нового входу без оновлення своїх ваг.

## Практичне застосування

- Демонструвати обов’язкову структуру виходу.
- Уточнювати крайові випадки класифікації чи видобування.
- Установлювати термінологію предметної області або правила позначення.
- Показувати оброблення відсутніх, неоднозначних або недійсних вхідних даних.

## Рекомендації з проєктування

Вибирайте репрезентативні, а не лише прості приклади. Додавайте позитивні й негативні випадки, коли відмінність важлива. Зберігайте послідовне форматування й перевіряйте, щоб приклади не містили випадкових інструкцій, які суперечать задачі. Для класифікації достатньо балансуйте мітки, щоб модель не засвоїла оманливу перевагу більшості.

## Компроміси й обмеження

Приклади займають контекст і можуть надто сильно прив’язати модель до поверхневих формулювань. Погані приклади здатні погіршити результат більше, ніж їх відсутність. Такий промптинг також не дає стійких знань і не гарантує однакової поведінки різних версій моделі.

## Типові помилки

- Надавати приклади, що суперечать письмовим інструкціям.
- Використовувати лише ідеальні випадки, пропускаючи важливі збої.
- Дозволяти моделі копіювати чутливі дані з демонстрацій.
- Припускати, що більша кількість прикладів завжди поліпшує результат.
