# Структурований вивід

Структурований вивід — це згенеровані моделлю дані у визначеному машинозчитуваному форматі: JSON, JSON Schema, XML, CSV або типізованому об’єкті аргументів виклику функції.

## Переклади

- [English](../../)
- Українська

## Основна ідея

Застосункам часто потрібні значення, які можна розібрати й перевірити, а не проза, що лише виглядає структурованою. Надійний структурований вивід поєднує чітку схему, підтримку моделі чи декодера, перевірку й оброблення помилок. Схема описує форму й типи, але не гарантує фактичної правильності значень.

## Практичне застосування

- Видобувати сутності, класифікації або параметри дій.
- Створювати вхідні дані для API та етапів робочого процесу.
- Забезпечувати обов’язкові поля й перелічувані значення.
- Відокремлювати машинозчитувані результати від пояснень для користувача.
- Перевіряти вихід перед виконанням будь-якої значущої дії.

## Міркування щодо реалізації

- Віддавайте перевагу нативній генерації за схемою, коли постачальник її підтримує.
- Зберігайте схеми малими й уникайте глибоко неоднозначних об’єднань.
- Визначайте необов’язкові поля свідомо, а не приймайте довільно відсутні дані.
- Відхиляйте або виправляйте некоректний вихід за обмеженою політикою повторів.
- Після синтаксичної перевірки застосовуйте предметну, наприклад для діапазонів дат або дозволених ідентифікаторів.

## Компроміси й обмеження

Суворі схеми підвищують надійність інтеграції, але можуть ускладнити подання нюансованих відповідей. Великі схеми займають контекст і можуть знижувати точність моделі. Синтаксично коректний об’єкт усе одно може містити вигадані ID, непідтверджені твердження чи небезпечні значення.

## Типові помилки

- Розбирати довільну прозу крихкими регулярними виразами.
- Виконувати згенеровані аргументи без перевірки.
- Вважати відповідність схемі перевіркою фактів.
- Вимагати водночас необмежену прозу й суворий JSON в одному каналі виходу.
