# Галюцинації

Галюцинація — це непідтверджений, неправильний або вигаданий вихід моделі, поданий у правдоподібній формі.

## Переклади

- [English](../../)
- Українська

## Основна ідея

Мовні моделі генерують імовірні послідовності токенів; вони не перевіряють автоматично кожне твердження за довіреним джерелом. Галюцинації можуть мати вигляд вигаданих фактів, посилань, методів API, цитат, шляхів до файлів, числових результатів або тверджень, що перебільшують непевні докази.

## Поширені причини

- Запитана інформація відсутня, неоднозначна або поза надійними знаннями моделі.
- Промпт заохочує відповідь навіть за відсутності доказів.
- Отриманий контекст неповний, застарілий або суперечливий.
- Від моделі вимагають точних ідентифікаторів чи цитат без доступу до джерела.
- Довгий контекст приховує найдоречніші докази.

## Зменшення ризику

- Використовуйте пошук, інструменти або авторитетні джерела даних для перевірних тверджень.
- Вимагайте посилань, які можна перевірити, а не лише тексту, схожого на посилання.
- Просіть модель розрізняти докази, висновки й невизначеність.
- Перевіряйте обчислення, URL, код і структуровані значення детермінованими системами.
- Оцінюйте частоту галюцинацій на репрезентативних задачах.

## Компроміси й обмеження

Нижча температура може підвищити послідовність, але не гарантує правдивості. RAG може зменшити непідтверджені відповіді, але внести помилки пошуку чи отруєні докази. Політики з частими відмовами можуть зменшити галюцинації ціною охоплення відповідей.

## Типові помилки

- Вважати впевнене формулювання ознакою правильності.
- Сприймати коректну відповідь JSON як перевірені дані.
- Просити модель «ніколи не галюцинувати», не надаючи доказів або перевірки.
