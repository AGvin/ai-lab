# Резидентність даних

Резидентність даних визначає географічне або юрисдикційне місце їх зберігання чи оброблення.

## Переклади

- [English](../../)
- Українська

## Основна ідея

Робочі процеси ШІ можуть передавати промпти, файли, ембединги, журнали й резервні копії між кількома сервісами та регіонами. Тому вимоги резидентності стосуються всього ланцюга, а не лише основної кінцевої точки моделі.

## Практичні запитання

- Де обробляються промпти й завантажені файли?
- Де зберігаються журнали, кеші, резервні копії й ембединги?
- Чи може персонал підтримки мати доступ з іншої юрисдикції?
- Які субпідрядники й регіони залучені?
- Чи дозволене регіональне аварійне перемикання?

## Компроміси й обмеження

Обмеження регіонів може зменшити вибір постачальників, доступність або продуктивність. Регіональна кінцева точка може залежати від глобальних систем керування, якщо договір явно не визначає межі.

## Типові помилки

- Плутати резидентність із приватністю чи суверенітетом даних.
- Перевіряти лише API моделі й ігнорувати телеметрію.
- Вважати адресу компанії місцем оброблення.
- Не документувати регіони резервування й аварійного відновлення.
