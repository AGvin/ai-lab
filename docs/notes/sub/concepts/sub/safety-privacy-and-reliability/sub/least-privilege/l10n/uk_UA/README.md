# Найменші привілеї

Принцип найменших привілеїв надає системі ШІ лише дозволи й доступ до даних, потрібні для поточної задачі.

## Переклади

- [English](../../)
- Українська

## Основна ідея

Здібна модель не повинна автоматично отримувати широкі операційні повноваження. Дозволи мають бути вузькими за областю, часом, ресурсом і дією. Можливості читати, готувати чернетку, схвалювати й виконувати слід за можливості розділяти.

## Практичні засоби контролю

- Використовуйте сервісні облікові записи для конкретних задач.
- Для аналізу віддавайте перевагу доступу лише для читання.
- Обмежуйте інструменти явними бізнес-операціями.
- Обмежуйте шляхи, репозиторії, одержувачів та орендарів.
- Використовуйте короткочасні облікові дані.
- Вимагайте схвалення для чутливих записів.

## Компроміси й обмеження

Детальні дозволи потребують більше налаштувань і можуть переривати робочі процеси. Однак ці витрати зазвичай нижчі за наслідки скомпрометованого чи помилкового агента з високими привілеями.

## Типові помилки

- Для зручності надавати агенту права адміністратора.
- Повторно використовувати необмежену сесію людини.
- Дозволяти універсальну оболонку, коли достатньо вузького API.
- Надавати постійний доступ для одноразової задачі.
