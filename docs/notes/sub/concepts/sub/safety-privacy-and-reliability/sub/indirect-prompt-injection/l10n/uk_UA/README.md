# Непряма ін’єкція у промпт

Непряма ін’єкція виникає, коли шкідливі або суперечливі інструкції вбудовані в зовнішній вміст, який читає система ШІ: вебсторінки, листи, документи, коментарі коду чи знайдені знання.

## Переклади

- [English](../../)
- Українська

## Основна ідея

Користувач може попросити агента узагальнити сторінку, але вона міститиме текст із вимогою розкрити дані чи викликати інструмент. Модель бачить інструкції задачі й сторінку як токени та може виконати недовірений текст, якщо застосунок не обмежує поведінку.

## Сценарії високого ризику

- Браузерні й поштові агенти.
- RAG над документами користувачів.
- Репозиторні агенти, що читають issues або коментарі.
- Конвеєри документів із доступом до інструментів.
- Мультимодальні системи, що читають інструкції із зображень.

## Зменшення ризику

Позначайте зовнішній вміст як недовірені дані, мінімізуйте інструменти під час читання, забезпечуйте авторизацію поза моделлю, фільтруйте знайдене й вимагайте схвалення перед побічними ефектами. Відокремлюйте видобування вмісту від виконання дій.

## Типові помилки

- Довіряти джерелу лише через внутрішній індекс.
- Дозволяти документам перевизначати роль агента.
- Надавати задачі узагальнення доступ на запис.
- Приховувати текст ін’єкції візуально й очікувати, що модель його проігнорує.
