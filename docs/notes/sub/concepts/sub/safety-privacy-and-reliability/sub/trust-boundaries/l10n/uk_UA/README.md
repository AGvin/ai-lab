# Межі довіри

Межа довіри розділяє компоненти, ідентичності, інструкції та дані з різними рівнями повноважень або надійності.

## Переклади

- [English](../../)
- Українська

## Основна ідея

Системи ШІ мають розрізняти політику застосунку, автентифікований намір користувача, результати інструментів, отримані документи, публічний вебвміст і згенерований моделлю текст. Перетин межі довіри потребує валідації, а не припущення, що весь контекст однаково авторитетний.

## Приклади

- Системна політика та input користувача.
- Дані автентифікованого облікового запису та публічні документи.
- Retrieval лише для читання та інструменти з можливістю запису.
- Пропозиції моделі та виконані операції.
- Дані одного tenant і контекст іншого.

## Практичне застосування

Представляйте довіру в архітектурі й коді, а не лише у формулюванні prompt. Додавайте до даних ідентичність і походження. Виконуйте авторизацію під час запуску інструмента. Вилучайте або ізолюйте схожий на інструкції вміст, коли компоненту потрібні лише факти.

## Компроміси й обмеження

Більша кількість меж підвищує складність реалізації та може зменшувати зручність workflow. Погано спроєктовані межі все одно можуть пропускати дані через журнали, кеші чи спільний стан.

## Типові помилки

- Сприйняття всього отриманого вмісту як довіреного.
- Дозвіл моделі вирішувати питання авторизації.
- Передавання необробленого output інструмента між tenants.
- Поєднання публічних даних і секретів в одному prompt без ізоляції.
