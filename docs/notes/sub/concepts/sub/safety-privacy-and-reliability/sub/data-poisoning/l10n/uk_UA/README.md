# Отруєння даних

Отруєння даних спотворює дані навчання чи адаптації для погіршення якості моделі, створення цільової поведінки або прихованих бекдорів.

## Переклади

- [English](../../)
- Українська

## Основна ідея

Зловмисник може додати неправильно позначені приклади, повторювану пропаганду, тригерні фрази, шкідливий код або непомітно змінені записи. Отруєння впливає на попереднє навчання, тонке налаштування, дані вподобань або набори оцінювання.

## Зменшення ризику

- Контролюйте, хто додає дані.
- Зберігайте походження джерел і підписи.
- Виявляйте дублікати, викиди й неочікувані шаблони міток.
- Перевіряйте нові чи високовпливові джерела.
- Розділяйте навчальні, валідаційні й тестові дані.
- Після підозри на компрометацію перебудовуйте й порівнюйте моделі.

## Компроміси й обмеження

Великі набори неможливо повністю перевірити вручну. Автоматичні фільтри можуть пропустити тонке отруєння або вилучити законні рідкісні випадки. Деякі атаки проявляються лише після розгортання.

## Типові помилки

- Довіряти відкритим внескам без відомостей про походження.
- Вважати згенеровані моделлю мітки беззаперечною істиною.
- Оцінювати даними з того самого скомпрометованого джерела.
- Вважати отруєння пошуку й навчання однією проблемою відновлення.
