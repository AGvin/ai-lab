# Отруєння retrieval

Отруєння retrieval маніпулює індексованими знаннями, щоб пошукова або RAG-система надавала оманливий, шкідливий чи інструктивний вміст.

## Переклади

- [English](../../)
- Українська

## Основна ідея

Зловмисник може додати документи з неправдивими твердженнями, оптимізованими ключовими словами, прихованим текстом або інструкціями prompt injection. Оскільки система отримує ці документи як свідчення, модель може повторювати їх або діяти за ними з видимим обґрунтуванням.

## Фактори ризику

- Контрольовані користувачами джерела ingestion.
- Автоматичне індексування без перевірки.
- Слабке відстеження походження та версій.
- Ранжування лише за подібністю.
- Агенти з інструментами, що діють за отриманими інструкціями.

## Зменшення ризику

Обмежуйте ingestion, перевіряйте ідентичність джерела, скануйте вміст, зберігайте походження, застосовуйте retrieval зі зважуванням довіри й відокремлюйте виділення фактів від виконання інструкцій. Оцінюйте retrieval на змагальних документах і суперечливих джерелах.

## Типові помилки

- Припущення, що результати внутрішнього пошуку довірені.
- Сприйняття наявності цитати як доказу якості джерела.
- Дозвіл отриманому тексту керувати інструментами.
- Невидалення скомпрометованих документів і похідних embeddings.
