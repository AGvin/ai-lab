# Ізоляція в sandbox

Sandboxing ізолює код, інструменти, файли або процеси, щоб збої та шкідлива поведінка мали обмежений вплив на host-систему.

## Переклади

- [English](../../)
- Українська

## Основна ідея

Sandbox обмежує такі ресурси, як шляхи файлової системи, мережевий доступ, створення процесів, пристрої, змінні середовища й час виконання. Це особливо важливо, коли модель генерує або вибирає код для виконання.

## Практичні засоби контролю

- Запускайте роботу в одноразових контейнерах або віртуальних машинах.
- Монтуйте лише потрібні файли.
- Вимикайте або обмежуйте вихідний мережевий доступ.
- Установлюйте обмеження CPU, пам’яті, диска й часу.
- Вилучайте облікові дані host і чутливі змінні середовища.
- Знищуйте середовище після завершення задачі.

## Компроміси й обмеження

Сильна ізоляція збільшує час запуску й може обмежити можливості інструментів. Самі лише контейнери не є досконалою межею безпеки для ворожого коду; для ризикованіших навантажень може знадобитися ізоляція у віртуальній машині або microVM.

## Типові помилки

- Запуск згенерованого коду безпосередньо на host.
- Монтування всього домашнього каталогу в контейнер.
- Збереження доступності хмарних облікових даних усередині sandbox.
- Необмежений мережевий доступ без моніторингу.
