# Рівні автономності

Рівні автономності описують, наскільки самостійно системі ШІ дозволено ухвалювати рішення й виконувати дії.

## Переклади

- [English](../../)
- Українська

## Приклад спектра

- **Дорадчий:** модель пропонує інформацію або дії, але не може їх виконувати.
- **Підготовка чернеток:** система готує артефакти для перевірки людиною.
- **Зі шлюзом схвалення:** вибрані дії виконуються лише після підтвердження.
- **Обмежено автономний:** агент діє самостійно в межах наперед визначених інструментів, бюджетів і політик.
- **Широко автономний:** система може планувати й діяти із застосуванням багатьох можливостей за обмеженого нагляду.

## Основна ідея

Автономність слід призначати окремо для кожної можливості й ризику, а не одним ярликом для всієї системи. Агент може автономно читати публічну документацію, але потребувати схвалення для зміни репозиторію чи надсилання електронної пошти.

## Практичне застосування

Вибирайте найнижчий рівень автономності, який усе ще дає корисну ефективність. Підвищуйте його лише після вимірювання надійності, додавання спостережуваності, звуження дозволів і визначення процедур відновлення.

## Компроміси й обмеження

Вища автономність зменшує обсяг ручної роботи, але посилює наслідки неправильних припущень, ін’єкцій у промпт, скомпрометованих інструментів і повторюваних побічних ефектів.

## Типові помилки

- Ототожнювати інтелект моделі з операційною довірою.
- Надавати однакову автономність діям читання й запису.
- Вилучати схвалення до вдосконалення перевірки.
- Не повідомляти користувачам фактичний рівень автономності.
