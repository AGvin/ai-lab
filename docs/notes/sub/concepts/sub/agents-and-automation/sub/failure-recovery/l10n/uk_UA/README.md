# Відновлення після збоїв

Відновлення після збоїв повертає робочий процес до відомого безпечного стану після відмов моделі, інструмента, мережі, інфраструктури або перевірки.

## Переклади

- [English](../../)
- Українська

## Основна ідея

Відновлення ширше за повторну спробу. Воно може продовжити роботу з контрольної точки, пропустити некритичний крок, перейти до резервного варіанта, компенсувати завершений побічний ефект, запросити втручання людини або безпечно зупинитися зі збереженням достатніх відомостей для подальшого продовження.

## Практичне проєктування

- Зберігайте контрольні точки після завершення важливих етапів.
- Класифікуйте збої як тимчасові, постійні, часткові або неоднозначні.
- Записуйте, які зовнішні ефекти точно відбулися.
- Застосовуйте компенсувальні дії, коли відкат неможливий.
- Зберігайте достатній діагностичний контекст для операторів.
- Визначайте кінцеві стани замість нескінченного циклу.

## Компроміси й обмеження

Надійне відновлення ускладнює робочий процес і потребує сховища. У розподілених системах може не бути єдиної авторитетної відповіді на питання, чи завершилася операція, для якої сплив час очікування.

## Типові помилки

- Перезапускати всю задачу після кожної помилки.
- Втрачати стан після завершення процесу.
- Припускати, що всі дії можна відкотити.
- Продовжувати після невдалої перевірки, не позначивши результат як небезпечний.
