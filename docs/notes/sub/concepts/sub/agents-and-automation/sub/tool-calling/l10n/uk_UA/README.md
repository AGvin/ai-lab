# Виклик інструментів

Виклик інструментів дає моделі змогу вибрати зареєстровану зовнішню операцію й сформувати аргументи для неї.

## Переклади

- [English](../../)
- Українська

## Основна ідея

Модель не виконує інструмент безпосередньо. Вона формує структурований запит, основний застосунок перевіряє й виконує його, а результат повертається моделі або робочому процесу. Це розділення необхідне, оскільки вихід моделі є недовіреними вхідними даними для рівня виконання.

## Типовий потік

1. Застосунок надає назви інструментів, описи та схеми аргументів.
2. Модель вибирає інструмент або відповідає без нього.
3. Застосунок перевіряє дозволи й аргументи.
4. Інструмент виконується в контрольованому середовищі.
5. Результат додається до стану або контексту.
6. Модель визначає, чи потрібна подальша робота.

## Практичне застосування

Виклик інструментів з’єднує моделі з пошуком, базами даних, календарями, електронною поштою, системами керування версіями, виконанням коду й внутрішніми сервісами. Інструменти мають бути вузькими, мати зрозумілі назви й відповідати бізнес-можливостям, а не надавати необмежений доступ до оболонки.

## Компроміси й обмеження

Описи інструментів займають контекст, а інструменти з перекривними функціями можуть ускладнювати вибір. Навіть коректні аргументи можуть бути семантично помилковими або небезпечними. Мережеві збої й часткові побічні ефекти потребують явної логіки відновлення.

## Типові помилки

- Виконувати аргументи моделі без перевірки.
- Надавати потужні універсальні інструменти замість обмежених операцій.
- Повертати моделі величезні невідфільтровані результати інструментів.
- Припускати, що виклик інструмента означає успішне виконання запитаної дії.
