# Перевірка й рефлексія

Перевірка визначає, чи відповідає результат доказам і критеріям прийняття. Рефлексія — це керований моделлю перегляд підходу, припущень або ймовірних помилок перед продовженням.

## Переклади

- [English](../../)
- Українська

## Основна ідея

Ці поняття пов’язані, але не рівнозначні. За можливості перевірка має використовувати об’єктивні засоби: тести, схеми, обчислення, порівняння з джерелами або правила дозволів. Рефлексія допомагає знаходити пропуски чи пропонувати виправлення, але залишається ще одним виходом моделі й може повторити початкову помилку.

## Практичне застосування

- Запускати тести після створення коду.
- Порівнювати твердження з отриманими доказами.
- Перевіряти структурований вивід за схемою та правилами предметної області.
- Просити другий прохід виявити пропущені випадки або суперечності.
- Вимагати людської перевірки для змін із великим впливом.

## Компроміси й обмеження

Додаткова перевірка збільшує затримку й вартість. Самоперевірка корелює з початковою генерацією та слабша за незалежні докази. Повторна рефлексія також може спричиняти непотрібні зміни вже правильного результату.

## Типові помилки

- Використовувати «перевір свою роботу» як єдиний засіб перевірки.
- Вважати другу відповідь моделі незалежним доказом.
- Не визначати, що саме означає успіх.
- Повторно генерувати результат замість аналізу детермінованих помилок тестів.
