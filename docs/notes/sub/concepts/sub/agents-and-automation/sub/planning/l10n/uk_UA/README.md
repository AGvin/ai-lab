# Планування

Планування — це процес вибору й упорядкування дій, потрібних для досягнення цілі за відомих обмежень.

## Переклади

- [English](../../)
- Українська

## Основна ідея

В агентних системах план може бути створений один раз, переглядатися після кожного спостереження або подаватися як граф залежностей. Добрі плани визначають передумови, очікувані результати, точки перевірки й умови, що потребують перепланування.

## Практичне застосування

- Розділяти зміну репозиторію на інспектування, реалізацію, тестування й публікацію.
- Визначати, яку інформацію потрібно отримати перед відповіддю.
- Планувати виклики інструментів, результати яких залежать від попередніх кроків.
- Виявляти дії, що потребують схвалення людини.

## Рекомендації з проєктування

Віддавайте перевагу коротким крокам, які можна перевірити, замість нечітких цілей. Відокремлюйте оборотне дослідження від виконання зі значущими наслідками. Додавайте критерії прийняття до кожного кроку й зберігайте поточний план у явному стані. Переплановуйте, коли докази спростовують припущення, замість примусового виконання початкової послідовності.

## Компроміси й обмеження

Докладне планування витрачає токени й може швидко застаріти в динамічних середовищах. Моделі можуть пропонувати правдоподібні, але неможливі кроки, якщо не знають інструментів або поточного стану.

## Типові помилки

- Вважати перший план незмінним.
- Планувати дії, які система насправді не може виконати.
- Пропускати кроки перевірки й відкату.
- Створювати довгий план для задачі з однією детермінованою операцією.
