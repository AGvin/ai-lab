# Агенти ШІ

Агент ШІ — це система, яка використовує модель для досягнення цілі через спостереження, рішення, виклики інструментів, зміни стану й повторювані дії.

## Переклади

- [English](../../)
- Українська

## Основна ідея

Модель стає частиною агента, коли здатна на більше, ніж створення однієї відповіді. Навколишній застосунок надає їй інструменти, робочий стан, потік керування, правила зупинки, дозволи та зворотний зв’язок від попередніх дій. Тому агентом є вся система, а не лише мовна модель.

## Типові складники

- Ціль або опис задачі.
- Модель, яка вибирає або пропонує дії.
- Інструменти на кшталт API, пошуку, виконання коду чи файлових операцій.
- Стан, який фіксує поступ і проміжні результати.
- Цикл, що спостерігає результати й визначає наступну дію.
- Політики схвалення, оброблення збоїв і завершення.

## Практичне застосування

Агенти корисні для задач, у яких наступна дія залежить від попередніх результатів: супроводу репозиторіїв, досліджень, адміністрування кількох систем, оброблення даних або підтримки клієнтів. За можливості залишайте детерміновані кроки поза моделлю й застосовуйте модель там, де потрібні інтерпретація або гнучкі рішення.

## Компроміси й обмеження

Більша автономність підвищує гнучкість, але також вартість, затримку й імовірність неправильних або небезпечних дій. Поведінку агента може бути складно відтворити, оскільки кожен результат інструмента впливає на подальші рішення.

## Типові помилки

- Називати агентом один запит до моделі.
- Надавати широкі облікові дані без контролю за принципом найменших привілеїв.
- Не визначати умови зупинки або максимальні бюджети дій.
- Сприймати згенеровані моделлю плани як довірені інструкції.
