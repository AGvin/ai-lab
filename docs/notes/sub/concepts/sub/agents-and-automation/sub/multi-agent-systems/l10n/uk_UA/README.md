# Мультиагентні системи

Мультиагентна система координує кількох агентів, які можуть спеціалізуватися, співпрацювати, конкурувати, делегувати або перевіряти одне одного.

## Переклади

- [English](../../)
- Українська

## Основна ідея

Кілька агентів корисні, коли задачі справді мають різні ролі, інструменти, дозволи або паралельні напрями роботи. Наприклад, агент-дослідник збирає докази, агент-реалізатор створює зміни, а рецензент перевіряє результат. Агентам усе одно потрібні спільний протокол, модель стану й механізм розв’язання конфліктів.

## Практичне застосування

- Паралельно аналізувати незалежні документи або репозиторії.
- Відокремлювати створення від перевірки.
- Призначати спеціалізованим ролям інструменти й промпти предметної області.
- Моделювати альтернативні пропозиції перед вибором однієї.

## Компроміси й обмеження

Мультиагентні системи збільшують кількість викликів моделей, накладні витрати координації та режими відмови. Агенти можуть підсилювати помилки одне одного, бо використовують подібні моделі чи контекст. Більша кількість агентів не створює незалежних доказів автоматично.

## Типові помилки

- Використовувати кількох агентів для задачі, яку можна виконати простим циклом.
- Надавати кожному агенту однакові роль та інформацію.
- Дозволяти агентам змінювати спільний стан без координації.
- Вважати згоду агентів доказом правильності.
