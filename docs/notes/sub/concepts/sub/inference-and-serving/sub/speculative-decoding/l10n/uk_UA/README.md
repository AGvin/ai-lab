# Спекулятивне декодування

Спекулятивне декодування прискорює авторегресійну генерацію: швидший чернетковий процес пропонує кілька токенів, які цільова модель перевіряє за меншу кількість послідовних кроків.

## Переклади

- [English](../../)
- Українська

## Основна ідея

Мала чернеткова модель прогнозує токени-кандидати. Велика цільова модель паралельно оцінює їх і приймає префікс, що відповідає її розподілу, зберігаючи розподіл виходу цільової моделі за припущень алгоритму. Відхилені токени повторно генерує цільова модель.

## Практичне застосування

- Зменшувати затримку великих мовних моделей.
- Використовувати меншу споріднену модель як чернеткову.
- Застосовувати пошук у промпті або n-грамне чернеткування, коли вихід повторює вхідний текст.
- Прискорювати інтерактивну генерацію без подальшого квантування цільової моделі.

## Компроміси й обмеження

Прискорення залежить від точності чернетки, вартості цільової моделі, використання обладнання й накладних витрат реалізації. Погано узгоджена чернеткова модель може майже не дати користі. Для завантаження чернеткової моделі або збереження її стану потрібна додаткова пам’ять.

## Типові помилки

- Припускати, що будь-яка мала модель придатна як чернеткова.
- Вимірювати лише частку прийнятих токенів, а не наскрізну затримку.
- Ігнорувати додаткові витрати пам’яті й завантаження.
- Очікувати однакового прискорення для всіх типів промптів і виходів.
