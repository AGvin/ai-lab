# Speculative Decoding

Speculative decoding accelerates autoregressive generation by using a faster draft process to propose several tokens that a target model verifies in fewer sequential steps.

## Core idea

A small draft model predicts candidate tokens. The larger target model evaluates those candidates in parallel and accepts the prefix that matches its own distribution, preserving the target model's output distribution under the algorithm's assumptions. Rejected tokens are regenerated by the target.

## Practical use

- Reduce latency for large language models.
- Reuse a smaller related model as a draft model.
- Apply prompt-lookup or n-gram drafting when output repeats input text.
- Improve interactive generation without quantizing the target model further.

## Trade-offs and limitations

Speedup depends on draft accuracy, target cost, hardware utilization, and implementation overhead. A poorly matched draft model may provide little benefit. Additional memory is required to load the draft model or maintain draft state.

## Common mistakes

- Assuming any small model is a suitable draft model.
- Measuring only accepted-token rate and not end-to-end latency.
- Ignoring the extra memory and loading cost.
- Expecting equal speedups for all prompt and output types.
