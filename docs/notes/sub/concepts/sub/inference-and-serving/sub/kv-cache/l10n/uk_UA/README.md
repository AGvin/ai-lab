# KV-кеш

KV-кеш зберігає ключі й значення уваги з раніше оброблених токенів, щоб авторегресійна модель не обчислювала їх заново для кожного нового токена.

## Переклади

- [English](../../)
- Українська

## Основна ідея

Під час генерації кожен новий токен звертає увагу на попередні. Кешування їхніх тензорів ключів і значень робить декодування практичним, але пам’ять кешу зростає з довжиною контексту, розміром пакета, кількістю шарів, розмірністю уваги й точністю кешу.

## Практичне застосування

- Оцінювати, скільки контексту вміститься після завантаження ваг моделі.
- Вибирати межі пакета й паралельності для сервера.
- Оцінювати формати кешу нижчої точності, коли вони підтримуються.
- Повторно використовувати стан промпту через кешування префікса або контексту.

## Компроміси й обмеження

Більший кеш забезпечує довший контекст або більше одночасних сесій, але споживає значний обсяг RAM чи VRAM. Квантовані кеші заощаджують пам’ять, але можуть впливати на якість. Архітектури з ковзним вікном і групованими запитами змінюють поведінку кешу, але не усувають його.

## Типові помилки

- Розраховувати місткість лише за вагами моделі.
- Тестувати короткий контекст, а розгортати довгий.
- Припускати, що заявлена межа контексту вміститься на будь-якому обладнанні.
- Плутати постійне кешування контексту з KV-кешем активного запиту в пам’яті.
