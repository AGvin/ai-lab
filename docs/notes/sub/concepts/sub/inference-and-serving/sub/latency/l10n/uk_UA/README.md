# Затримка

Затримка — це час, потрібний системі інференсу для досягнення визначеного етапу відповіді.

## Переклади

- [English](../../)
- Українська

## Корисні показники затримки

- **Затримка в черзі:** час очікування до початку оброблення.
- **Затримка завантаження моделі:** час підготовки моделі до роботи.
- **Час до першого токена:** затримка до повернення першого згенерованого токена.
- **Міжтокенна затримка:** час між згенерованими токенами.
- **Наскрізна затримка:** загальний час до отримання повної відповіді.

## Основна ідея

Одне середнє значення приховує важливі відмінності користувацького досвіду. Для інтерактивного чату важливі час до першого токена й плавне надходження токенів, тоді як пакетне оброблення може надавати пріоритет загальному часу завершення.

## Практичне застосування

Вимірюйте перцентилі p50, p95 і p99 за реалістичної паралельності. Відокремлюйте час оброблення промпту від декодування. Під час оцінювання застосунку враховуйте затримки мережі, черги, пошуку й інструментів.

## Компроміси й обмеження

Пакетне оброблення може підвищити пропускну здатність, збільшуючи затримку окремого запиту. Більші моделі й довші промпти зазвичай збільшують затримку. Кешування зменшує повторну роботу, але додає витрати пам’яті й складність інвалідації.

## Типові помилки

- Повідомляти лише найшвидший ізольований запит.
- Використовувати середню затримку без хвостових перцентилів.
- Вилучати час черги й пошуку з показників для користувача.
- Порівнювати затримку для різної довжини виходу.
