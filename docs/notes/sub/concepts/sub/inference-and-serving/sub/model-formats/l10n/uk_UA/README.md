# Формати моделей

Формати моделей визначають, як зберігаються й передаються ваги, метадані, файли токенізатора, графи обчислень і конфігурація.

## Переклади

- [English](../../)
- Українська

## Поширені ролі форматів

- **Safetensors:** безпечний формат серіалізації тензорів, поширений для ваг моделей.
- **GGUF:** самодостатній формат для локальних середовищ, сумісних із llama.cpp, який часто містить квантовані ваги й метадані.
- **ONNX:** формат обміну графами для сумісних середовищ інференсу.
- **Контрольні точки фреймворків:** формати, прив’язані до бібліотек на кшталт PyTorch або TensorFlow.

## Основна ідея

Репозиторій моделі часто містить кілька файлів: ваги, словник токенізатора, конфігурацію архітектури, шаблон чату, типові параметри генерації й необов’язкові адаптери. Сумісність залежить від усіх них, а не лише від розширення файла ваг.

## Практичне застосування

Вибирайте формат, який підтримують цільове середовище виконання й обладнання. До завантаження чи перетворення перевіряйте підтримку архітектури, сумісність токенізатора, тип квантування, конфігурацію контексту й ліцензію.

## Компроміси й обмеження

Перетворення може втратити метадані, внести зміни квантування або створити формат, непідтримуваний старішими середовищами. Самодостатні формати спрощують розгортання, але можуть дублювати великі файли ваг для різних квантувань.

## Типові помилки

- Завантажувати ваги з неправильним токенізатором або шаблоном чату.
- Припускати, що кожне середовище однаково підтримує кожну архітектуру в тому самому форматі.
- Вважати розширення файла достатньою інформацією про сумісність.
- Поширювати перетворені ваги без перевірки початкової ліцензії.
