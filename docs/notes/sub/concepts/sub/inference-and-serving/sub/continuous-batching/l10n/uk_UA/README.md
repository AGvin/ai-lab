# Безперервне пакетування

Безперервне пакетування динамічно об’єднує активні запити інференсу, щоб нова робота входила до пакета в міру завершення інших запитів.

## Переклади

- [English](../../)
- Українська

## Основна ідея

Статичне пакетування чекає фіксованої групи й обробляє її разом, марнуючи ресурси за різної довжини послідовностей. Безперервне пакетування планує токени або блоки послідовностей між запитами, підтримуючи зайнятість прискорювача й дозволяючи запитам починатися та завершуватися незалежно.

## Практичне застосування

- Багатокористувацькі API мовних моделей.
- Високопродуктивне обслуговування зі змінною довжиною виходу.
- Краще використання прискорювача під сталим навантаженням.
- Поєднання потокових відповідей із пакетними обчисленнями.

## Компроміси й обмеження

Планування додає складності й може впливати на справедливість. Великі активні пакети підвищують пропускну здатність, але можуть збільшити час до першого токена. Обмеженням може стати тиск на пам’ять від численних KV-кешів.

## Належна практика

Установлюйте межі приймання за токенами й пам’яттю, а не лише кількістю запитів. Вимірюйте перцентилі затримки за реалістичної паралельності. Захищайте короткі інтерактивні запити від голодування через дуже довгі генерації.

## Типові помилки

- Повідомляти пакетну пропускну здатність без користувацької затримки.
- Дозволяти необмежені довжини контексту й виходу.
- Ігнорувати фрагментацію пам’яті та місткість KV-кешу.
- Застосовувати одну політику планування до пакетних та інтерактивних навантажень.
