# Векторний пошук

Векторний пошук знаходить елементи, чиї vectors embeddings найближчі до вектора запиту за метрикою подібності або відстані.

## Переклади

- [English](../../)
- Українська

## Основна ідея

Точний пошук найближчих сусідів порівнює запит із кожним вектором і стає дорогим у великому масштабі. Індекси approximate nearest-neighbor обмінюють невелике зниження recall на значно швидший пошук. Поширені сімейства індексів організовують вектори у графи, кластери чи стиснені partitions.

## Практичні міркування

- Вибирайте метрику, сумісну зі способом навчання embedding-моделі.
- Вимірюйте recall відносно baseline точного пошуку на репрезентативних даних.
- Налаштовуйте кількість кандидатів, параметри індексу й використання пам’яті разом.
- Застосовуйте metadata й authorization filters до повернення вмісту.
- Перебудовуйте або мігруйте індекси зі зміною розмірності векторів чи моделей.

## Компроміси й обмеження

Наближені індекси можуть пропускати справді найближчі елементи. Вектори високої розмірності споживають багато пам’яті. Пошук подібності також повертає математично близьке, що не завжди корисне для кінцевої задачі.

## Типові помилки

- Використання Euclidean distance, коли модель очікує normalized cosine similarity.
- Порівняння оцінок різних індексів як каліброваних.
- Вибір налаштувань індексу лише за синтетичними benchmarks.
- Ігнорування видалення, версіонування чи очищення застарілих векторів.
