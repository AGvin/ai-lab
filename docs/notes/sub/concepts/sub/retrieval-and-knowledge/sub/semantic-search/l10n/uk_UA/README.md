# Семантичний пошук

Семантичний пошук отримує вміст через порівняння вивчених представлень значення, а не лише точних збігів слів.

## Переклади

- [English](../../)
- Українська

## Основна ідея

Запит і кандидатні документи перетворюються на vectors embeddings, після чого векторний індекс знаходить близькі елементи. Це дає змогу виконувати retrieval, коли користувач і джерело використовують різні формулювання, абревіатури чи перефразування.

## Практичні переваги

- Запитання природною мовою до документів.
- Cross-lingual або насичений перефразуваннями retrieval.
- Пошук концептуально пов’язаного коду, tickets або нотаток.
- Зіставлення наміру користувача з продуктами чи статтями підтримки.

## Компроміси й обмеження

Семантичний пошук може пропустити точні ідентифікатори, рідкісні назви, номери версій або заперечні фрази. Він також може отримувати тематично подібні уривки без потрібної відповіді. Результати сильно залежать від embedding-моделі, chunking і домену.

## Належна практика

Поєднуйте семантичний пошук із metadata filters і, коли важливі точні терміни, лексичним пошуком. Оцінюйте recall за кількох значень кількості кандидатів, а не лише перший результат. Використовуйте reranking, якщо початкова векторна подібність надто груба.

## Типові помилки

- Сприйняття найближчих сусідів як перевірених відповідей.
- Використання лише семантичного пошуку для SKU, кодів помилок або юридичних цитат.
- Порівняння оцінок подібності між непов’язаними моделями чи індексами.
- Ігнорування контролю доступу під час retrieval.
