# GraphRAG

GraphRAG — підхід Retrieval-Augmented Generation, що використовує сутності, зв’язки, спільноти чи обхід графа для формування контексту.

## Переклади

- [English](../../)
- Українська

## Основна ідея

Звичайний RAG зазвичай отримує незалежні текстові chunks. GraphRAG додає структуру, здатну поєднати інформацію з різних документів: людей із проєктами, сервіси з інцидентами або поняття через цитування. Граф можна створити з наявного графа знань або виділити з тексту.

## Практичне застосування

- Запитання, що потребують багатокрокових зв’язків.
- Огляди багатьох пов’язаних сутностей або документів.
- Дослідження залежностей, власності чи причинних ланцюгів.
- Навігація організаційними або технічними знаннями.

## Компроміси й обмеження

Побудова графа дорога й схильна до помилок. Виділення сутностей, дедуплікація й типізація зв’язків можуть додавати хибні зв’язки. Обхід графа також може отримувати структурно пов’язану, але семантично нерелевантну інформацію.

## Належна практика

Зберігайте вихідні свідчення для кожного вузла й ребра. Оцінюйте виділення графа окремо від генерації відповіді. Використовуйте retrieval графа лише там, де зв’язки додають цінність; прості фактичні запити можуть краще працювати зі звичайним гібридним пошуком.

## Типові помилки

- Сприйняття автоматично виділених зв’язків як перевірених фактів.
- Побудова графа без класу запитань, якому він потрібен.
- Втрата походження документа після об’єднання сутностей.
- Припущення, що обхід графа замінює лексичний і семантичний retrieval.
