# BM25

BM25 — імовірнісна функція лексичного ранжування для оцінювання відповідності документа термінам запиту.

## Переклади

- [English](../../)
- Українська

## Основна ідея

BM25 винагороджує документи з важливими термінами запиту, контролюючи частоту термінів і довжину документа. Термін, присутній у небагатьох документах, отримує більшу вагу, ніж поширений. Повторення терміна допомагає лише до точки насичення, що не дає простій частоті домінувати в ранжуванні.

## Практичне застосування

- Пошук точної технічної термінології.
- Retrieval повідомлень про помилки, ідентифікаторів та іменованих сутностей.
- Надання набору лексичних кандидатів для гібридного пошуку.
- Створення сильного й недорогого retrieval baseline.

## Важливі параметри

- `k1` контролює швидкість насичення користі від частоти терміна.
- `b` контролює нормалізацію довжини документа.

Дефолтні значення часто прийнятні, але analyzers, ваги полів, stop words і структура документа зазвичай важливіші за малі зміни параметрів.

## Компроміси й обмеження

BM25 не розуміє перефразувань або ширшої семантичної подібності без розширення запиту чи синонімів. Він також може завищувати вагу рідкісних термінів, які насправді не релевантні.

## Типові помилки

- Налаштування параметрів до виправлення tokenization і дизайну полів.
- Одне об’єднане поле для заголовків, основного тексту й ідентифікаторів.
- Припущення, що оцінки BM25 можна порівнювати між різними запитами.
- Відмова від BM25 через кращий результат embeddings на малому demo-наборі.
