# Обґрунтування

Обґрунтування пов’язує згенеровані твердження з наданими свідченнями, результатами інструментів або авторитетними даними замість покладання лише на внутрішні статистичні знання моделі.

## Переклади

- [English](../../)
- Українська

## Основна ідея

Обґрунтована відповідь має підтверджуватися інформацією, доступною системі під час генерації. Для обґрунтування можна використовувати отримані документи, записи бази даних, обчислення, виконання коду, sensors або API. Застосунок має зберігати зв’язок між твердженнями та свідченнями, щоб результат можна було перевірити.

## Практичне застосування

- Відповідати на запитання за внутрішньою документацією.
- Отримувати актуальні дані з API або баз даних.
- Створювати підсумки з посиланнями на джерела.
- Використовувати калькулятори чи код для точних обчислень.
- Обмежувати рекомендації перевіреними даними каталогу.

## Компроміси й обмеження

Якість обґрунтування залежить від якості свідчень. Неправильні, застарілі чи шкідливі джерела можуть створювати впевнено обґрунтовані, але хибні відповіді. Модель також може цитувати релевантний уривок, який не підтверджує конкретного твердження.

## Належна практика

Виконуйте вузький retrieval, записуйте походження й розрізняйте прямі свідчення та висновки. Перевіряйте критичні значення поза моделлю. Дозволяйте системі повідомляти про недостатність свідчень.

## Типові помилки

- Визнання відповіді обґрунтованою лише тому, що документи додано до контексту.
- Прийняття наявності цитати без перевірки її підтвердної сили.
- Змішування довірених даних із недовіреними інструкціями.
- Приховування конфліктів джерел замість повідомлення про них.
