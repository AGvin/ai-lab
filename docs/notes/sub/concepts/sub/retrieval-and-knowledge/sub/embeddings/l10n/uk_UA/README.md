# Embeddings

Embeddings — числові вектори, що представляють семантичні або структурні властивості тексту, зображень, аудіо, користувачів, продуктів чи інших об’єктів.

## Переклади

- [English](../../)
- Українська

## Основна ідея

Embedding-модель перетворює input на вектор фіксованої довжини. Елементи з подібним значенням або використанням зазвичай розміщуються ближче за мірою відстані, наприклад cosine similarity або dot product. Сам вектор не є зрозумілим людині поясненням; це вивчене представлення, оптимізоване для певних задач і навчальних даних.

## Практичне застосування

- Семантичний пошук документів і коду.
- Retrieval-Augmented Generation.
- Кластеризація й виявлення дублікатів.
- Рекомендації та зіставлення за подібністю.
- Класифікація за найближчими прикладами або легкими моделями.

## Проєктні міркування

Використовуйте однакову embedding-модель і preprocessing для індексованих документів та вхідних запитів. Записуйте версію моделі й розмірність вектора. Оцінюйте retrieval на доменних запитах, адже загалом потужна embedding-модель може погано працювати з кодами продуктів, юридичною термінологією чи вихідним кодом.

## Компроміси й обмеження

Embeddings стискають інформацію й можуть втрачати точні формулювання, дати, ідентифікатори чи заперечення. Зміна embedding-моделі зазвичай потребує повторного створення embeddings індексу. Оцінки подібності не є каліброваними ймовірностями й не повинні тлумачитися як упевненість без оцінювання.

## Типові помилки

- Змішування векторів несумісних embedding-моделей.
- Використання семантичного retrieval для точних ID без лексичного пошуку.
- Припущення, що висока оцінка подібності доводить фактичну релевантність.
- Ігнорування впливу мови, домену й розміру chunk.
