# Фільтрація metadata

Фільтрація metadata обмежує retrieval структурованими атрибутами, як-от джерело, дата, мова, tenant, продукт, версія документа чи політика доступу.

## Переклади

- [English](../../)
- Українська

## Основна ідея

Сама семантична чи лексична подібність не може виразити кожну вимогу retrieval. Фільтри зменшують простір кандидатів до або під час пошуку й допомагають гарантувати належність результатів правильному користувачу, проєкту, часовому діапазону чи класу документів.

## Практичне застосування

- Забезпечувати межі tenant і репозиторію.
- Отримувати лише актуальні версії документів.
- Обмежувати результати за продуктом, локаллю чи типом вмісту.
- Виключати архівні або несхвалені джерела.
- Шукати у вибраній користувачем колекції.

## Проєктні міркування

Розрізняйте зручні пошукові фільтри й безпекову авторизацію. Чутливі для безпеки фільтри мають примусово застосовуватися рівнем даних і не залежати від згенерованих моделлю metadata. Нормалізуйте значення metadata й визначайте поведінку оновлення зі зміною вихідних документів.

## Компроміси й обмеження

Суворі фільтри можуть знижувати recall, коли metadata відсутні або неправильні. Складні вирази фільтрів також можуть обмежувати продуктивність індексу чи поводитися по-різному у векторних базах даних.

## Типові помилки

- Дозвіл моделі вибирати необмежений tenant ID.
- Зберігання неузгоджених дат або міток категорій.
- Застосування фільтрів після повернення неавторизованого вмісту.
- Неоновлення metadata після заміни документів.
