# Векторні бази даних

Векторна база даних зберігає embeddings і пов’язані metadata, будує індекси подібності й надає операції додавання, оновлення, фільтрації та retrieval векторів.

## Переклади

- [English](../../)
- Українська

## Основна ідея

База даних — інфраструктура навколо векторного пошуку. Корисні можливості включають approximate nearest-neighbor indexing, metadata filters, namespaces або tenants, persistence, replication, видалення та інтеграцію з pipelines ingestion документів.

## Критерії вибору

- Підтримувані розмірності векторів і метрики відстані.
- Поведінка фільтрації й інтеграція контролю доступу.
- Характеристики побудови та оновлення індексу.
- Вимоги до пам’яті, сховища й експлуатації.
- Backup, replication і disaster recovery.
- Підтримка гібридного лексичного й векторного retrieval.

## Компроміси й обмеження

Спеціалізована векторна база даних може спростити масштабування й експлуатацію, але додає ще одну систему для обслуговування. Звичайних баз даних із векторними extensions може бути достатньо за помірного обсягу даних, затримки та частоти оновлень. База даних не компенсує погані embeddings, chunking чи оцінювання.

## Типові помилки

- Вибір бази даних до вимірювання вимог навантаження.
- Зберігання векторів без source IDs, версій або шляхів видалення.
- Сприйняття tenant metadata як заміни примусовій авторизації.
- Припущення, що швидший пошук найближчих сусідів гарантує кращі RAG-відповіді.
