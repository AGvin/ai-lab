# Щільні й розріджені моделі

Щільні моделі активують більшість параметрів для кожного входу, а розріджені — лише вибрані параметри або компоненти.

## Переклади

- [English](../../)
- Українська

## Основна ідея

Щільний трансформер виконує кожен блок прямого поширення для кожного токена. Розріджена архітектура на кшталт суміші експертів може спрямовувати кожен токен через малу підмножину експертів. Це дає велику загальну кількість параметрів без використання кожного параметра для кожного токена.

## Практичні наслідки

- Загальна кількість параметрів описує вимоги до сховища.
- Кількість активних параметрів краще наближує обсяг обчислень на токен.
- Маршрутизація між експертами створює вимоги до обміну даними й балансування навантаження.
- Розрідженим моделям може бути потрібна спеціалізована підтримка середовища виконання.

## Компроміси й обмеження

Розріджені моделі можуть забезпечувати більшу місткість за подібного обсягу обчислень, але їх складно ефективно навчати й обслуговувати. Ваги всіх експертів усе одно можуть потребувати зберігання в пам’яті. Нерівномірна маршрутизація може перевантажувати одних експертів, залишаючи інших бездіяльними.

## Типові помилки

- Безпосередньо порівнювати загальну кількість параметрів розрідженої моделі з активними параметрами щільної.
- Припускати, що велика MoE-модель уміститься як щільна модель із тією самою кількістю активних параметрів.
- Ігнорувати витрати обміну даними між пристроями.
- Ототожнювати розрідженість із квантуванням або проріджуванням.
