# Увага

Увага — це механізм, який обчислює, наскільки сильно одне подання має використовувати інформацію з інших подань.

## Переклади

- [English](../../)
- Українська

## Основна ідея

Запит порівнюється з ключами для отримання оцінок уваги. Оцінки нормалізуються й використовуються для поєднання відповідних значень. У трансформерах численні голови уваги паралельно засвоюють різні шаблони взаємодії.

## Практичні ролі

- Пов’язувати згенерований токен із доречними попередніми токенами.
- Узгоджувати текст із ділянками зображення або ознаками аудіо.
- Поєднувати інформацію енкодера під час генерації послідовності.
- Подавати далекі залежності безпосередніше, ніж проста рекурентність.

## Компроміси й обмеження

Стандартна повна увага порівнює багато пар позицій і стає дорогою для довгих послідовностей. Оцінки уваги — це внутрішні обчислення, а не калібровані докази причинної важливості чи зрозуміле людині міркування.

## Типові помилки

- Тлумачити найбільшу вагу уваги як пояснення моделі.
- Припускати, що сама увага зберігає постійну пам’ять.
- Ігнорувати маскування й позиційне кодування.
- Вважати всі реалізації уваги обчислювально однаковими.
