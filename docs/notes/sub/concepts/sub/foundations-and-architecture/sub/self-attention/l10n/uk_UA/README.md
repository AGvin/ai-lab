# Самоувага

Самоувага — це увага, у якій запити, ключі й значення походять із позицій однієї послідовності або набору подань.

## Переклади

- [English](../../)
- Українська

## Основна ідея

Кожна позиція може поєднувати інформацію з інших позицій відповідно до засвоєних оцінок доречності. У каузальних мовних моделях маскування не дає токену звертати увагу на майбутні токени під час генерації. Двонапрямні енкодери можуть дозволяти увагу в обох напрямках.

## Практичне значення

Самоувага дає моделям змогу пов’язувати займенники з попередніми іменниками, зіставляти посилання в різних частинах функції коду або поєднувати віддалені фрагменти документа. Кілька шарів поступово перетворюють ці зв’язки на подання вищого рівня.

## Компроміси й обмеження

Повна самоувага стає дорогою зі зростанням довжини послідовності. Розріджені механізми, ковзні вікна або групування зменшують вартість, але змінюють позиції, які взаємодіють безпосередньо. Увага також може зосереджуватися на статистично корисних шаблонах, яким не можна довіряти семантично.

## Типові помилки

- Забувати про різницю між каузальним і двонапрямним маскуванням.
- Припускати, що кожен токен однаково звертається до всього заявленого контексту.
- Вважати карти уваги остаточним засобом інтерпретації.
- Плутати самоувагу з перехресною увагою між модальностями чи послідовностями.
