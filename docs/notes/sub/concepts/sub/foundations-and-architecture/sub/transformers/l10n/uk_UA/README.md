# Трансформери

Трансформер — це архітектура нейронної мережі, побудована навколо уваги, шарів прямого поширення, залишкових з’єднань і нормалізації.

## Переклади

- [English](../../)
- Українська

## Основна ідея

Трансформери паралельно обробляють подання токенів під час навчання й оброблення промпту. Увага дає кожній позиції змогу поєднувати інформацію з інших доречних позицій, а шари прямого поширення незалежно перетворюють кожну позицію. Позиційна інформація зберігає порядок послідовності.

## Поширені форми

- Моделі лише з енкодером для подань і класифікації.
- Моделі лише з декодером для авторегресійної генерації.
- Моделі енкодер-декодер для задач «послідовність у послідовність».
- Мультимодальні трансформери, що інтегрують візуальні або аудіотокени.

## Практичне застосування

Трансформери лежать в основі більшості сучасних мовних моделей і багатьох систем зору, аудіо та мультимодальності. На вимоги середовища виконання впливають кількість шарів, розміри прихованих подань, конфігурація уваги, розмір словника й довжина контексту.

## Компроміси й обмеження

Вартість стандартної уваги стрімко зростає з довжиною послідовності. Тому довгі контексти потребують значних обчислень і пам’яті KV-кешу. Трансформери також можуть засвоювати статистичні скорочені шляхи й самі по собі не забезпечують перевірки фактів чи постійної пам’яті.

## Типові помилки

- Вважати трансформер синонімом LLM.
- Припускати, що ваги уваги дають повне пояснення.
- Ігнорувати варіанти архітектури під час порівняння кількості параметрів.
- Очікувати, що довший контекст гарантує краще пригадування.
