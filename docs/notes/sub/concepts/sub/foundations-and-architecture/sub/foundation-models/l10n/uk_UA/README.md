# Фундаментальні моделі

Фундаментальна модель — це широко навчена модель, здатна підтримувати багато подальших задач через промптинг, пошук, інструменти, тонке налаштування або адаптери.

## Переклади

- [English](../../)
- Українська

## Основна ідея

Замість навчання окремої моделі з нуля для кожної задачі фундаментальна модель засвоює придатні до повторного використання подання з великих і різноманітних даних. Поширені приклади — мовні моделі, візуально-мовні моделі й універсальні моделі генерації зображень.

## Методи адаптації

- Промптинг без прикладів і з кількома прикладами.
- Системні промпти та структурований вивід.
- Генерація, доповнена пошуком.
- Виклики інструментів і агентні робочі процеси.
- Тонке налаштування, LoRA або інші адаптери.

## Практичне застосування

Фундаментальні моделі знижують вартість створення нових застосунків, оскільки одна модель може підтримувати узагальнення, видобування, програмування, пошук, генерацію та багато інших задач. Під час вибору слід враховувати можливості, ліцензію, приватність, варіанти розгортання й результати оцінювання.

## Компроміси й обмеження

Широкі можливості не гарантують точності в предметній області. Дані попереднього навчання можуть бути застарілими, упередженими, неповними або юридично обмеженими. Великі моделі також потребують значних ресурсів для інференсу або залежності від постачальника.

## Типові помилки

- Припускати, що універсальна модель не потребує оцінювання для конкретної задачі.
- Виконувати тонке налаштування до перевірки промптингу й пошуку.
- Ігнорувати ліцензії моделі та наборів даних.
- Вважати модель завершеним застосунком без засобів контролю чи інструментів.
