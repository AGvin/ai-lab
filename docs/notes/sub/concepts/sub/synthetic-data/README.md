# Synthetic Data

Synthetic data is training or evaluation data generated by rules, simulations, procedural systems, or other models rather than collected directly from real events.

## Core idea

Synthetic generation can expand coverage, create rare edge cases, or avoid exposing real personal data. It is most useful when generation rules are controllable and outputs are validated against independent criteria.

## Practical use

- Produce instruction-response candidates for review.
- Simulate failures or rare operational scenarios.
- Balance classes that are underrepresented in real data.
- Create privacy-preserving prototypes.
- Generate test cases with known expected structure.

## Trade-offs and limitations

Synthetic data inherits the generator's errors and biases. Repeated model-generated patterns can narrow diversity or cause a model to imitate artifacts rather than the real domain. Synthetic data is not automatically anonymous if it reproduces memorized information.

## Common mistakes

- Training directly on unreviewed model output.
- Using the same model to generate and judge all examples.
- Replacing real validation data entirely with synthetic data.
- Measuring dataset size instead of coverage and correctness.
