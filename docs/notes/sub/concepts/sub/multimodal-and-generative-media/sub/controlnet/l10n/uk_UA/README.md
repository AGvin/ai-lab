# ControlNet

ControlNet зумовлює дифузійну генерацію структурною інформацією: краями, глибиною, позою, сегментацією, лінійним рисунком або нормалями поверхонь.

## Переклади

- [English](../../)
- Українська

## Основна ідея

Керувальна модель обробляє структурну карту й додає її настанови до дифузійної мережі. Це дає промпту визначати вигляд, а керувальному сигналу — обмежувати макет або геометрію. Різні моделі ControlNet навчають для різних типів карт і сімейств базових моделей.

## Практичне застосування

- Зберігати позу чи композицію референсу.
- Перетворювати ескізи або лінійні рисунки на рендери.
- Зберігати геометрію приміщення за картами глибини.
- Дотримуватися структури країв зі зміною стилю.
- Створювати сталі макети за картами сегментації.

## Компроміси й обмеження

Сильний контроль може зменшити творчість або конфліктувати з промптом. Слабкий контроль може не зберегти структуру. Помилки препроцесора стають обмеженнями генерації, а несумісні сімейства моделей дають погані результати.

## Типові помилки

- Використовувати керувальну модель для іншої базової моделі.
- Застосовувати надмірну силу контролю на всіх кроках видалення шуму.
- Сприймати шумну карту глибини чи пози як точну геометрію.
- Поєднувати кілька контролів без балансування їхнього впливу.
