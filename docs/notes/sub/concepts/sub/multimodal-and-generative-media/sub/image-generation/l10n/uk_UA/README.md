# Генерація зображень

Генерація зображень створює нові зображення з тексту, референсів, масок, макетів, карт глибини, поз або інших умовних сигналів.

## Переклади

- [English](../../)
- Українська

## Основна ідея

Сучасні системи зазвичай використовують дифузійні чи споріднені генеративні архітектури, які починають із шуму або латентного подання й поступово будують зображення. Промпт та умови спрямовують композицію, об’єкт, стиль, освітлення й інші властивості, але генерація залишається ймовірнісною.

## Практичне застосування

- Концепт-арт і пошук візуальних ідей.
- Матеріали для маркетингу й соціальних мереж.
- Макети продуктів і розкадрування.
- Кероване редагування й відновлення зображень.
- Синтетичні візуальні набори даних.

## Компроміси й обмеження

Моделі можуть створювати анатомічні помилки, непослідовний текст, дубльовані об’єкти або деталі, що не відповідають промпту. Точна ідентичність, логотипи, типографіка й сталість між кількома зображеннями часто потребують референсного зумовлення чи ручного редагування.

## Типові помилки

- Очікувати, що довгий промпт незалежно контролюватиме кожну деталь.
- Використовувати згенеровані зображення як фактичні докази.
- Ігнорувати ліцензії моделі, набору даних і виходу.
- Публікувати без перевірки рук, тексту, віддзеркалень і повторених об’єктів.
