# Оптимізація переваг

Оптимізація переваг коригує поведінку моделі за допомогою порівнянь або оцінок, що вказують, яким відповідям віддають перевагу.

## Переклади

- [English](../../)
- Українська

## Основна ідея

Supervised fine-tuning навчає модель імітувати цільові відповіді. Натомість оптимізація переваг використовує відносні судження, наприклад що одна відповідь корисніша, безпечніша, стисліша чи точніша за іншу. Методи включають RLHF, DPO та споріднені objectives.

## Практичне застосування

- Покращити виконання інструкцій і користь у розмові.
- Сформувати поведінку відмов і безпеки.
- Зменшити небажані стилі або багатослівність.
- Узгодити output зі спеціалізованими критеріями якості.

## Компроміси й обмеження

Мітки переваг суб’єктивні й можуть кодувати упередження annotator або політики. Оптимізація наближеної оцінки переваг може знизити різноманітність, спричинити надмірні відмови або винагороджувати переконливе формулювання замість фактичної правильності.

## Типові помилки

- Сприйняття переваг як об’єктивної істини.
- Збір порівнянь без чітких критеріїв.
- Використання однієї моделі-судді як єдиного джерела міток.
- Оцінювання лише частки перемог за перевагами з ігноруванням фактичних або task-метрик.
