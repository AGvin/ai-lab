# DPO

Direct Preference Optimization, або DPO, навчає модель безпосередньо на парах бажаних і відхилених відповідей без попереднього налаштування окремої reward model і повного циклу reinforcement learning.

## Переклади

- [English](../../)
- Українська

## Основна ідея

DPO збільшує відносну ймовірність бажаних відповідей, водночас обмежуючи адаптовану модель відносно reference model. Він спрощує навчання за перевагами й часто легший у реалізації, ніж RLHF.

## Практичне застосування

- Уточнити instruction-tuned модель за даними порівнянь.
- Скоригувати стиль, поведінку відмов або task-переваги.
- Експериментувати з preference alignment через простіший pipeline.

## Компроміси й обмеження

DPO усе одно залежить від якісних preference pairs і належної reference model. Він може надмірно підлаштуватися під артефакти анотації, зменшити різноманітність відповідей або оптимізувати суб’єктивний стиль замість правильності.

## Типові помилки

- Використання слабких або неузгоджених відхилених відповідей.
- Припущення, що DPO усуває потребу в оцінюванні.
- Порівняння DPO й RLHF без однакових даних і цілей.
- Навчання на preference pairs із прихованими фактичними помилками.
