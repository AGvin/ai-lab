# Adapters

Adapters — невеликі навчувані компоненти, приєднані до замороженої або переважно замороженої базової моделі для повторно використовуваної спеціалізації.

## Переклади

- [English](../../)
- Українська

## Основна ідея

Adapter додає параметри до вибраних шарів моделі або змінює їхні обчислення. Для різних задач можна завантажувати різні adapters без зберігання повної копії моделі. LoRA — один із широко вживаних методів у стилі adapter, але методи adapters можуть використовувати інші архітектури.

## Практичне застосування

- Підтримувати спеціалізовані варіанти моделі.
- Перемикати доменну поведінку під час роботи.
- Поширювати малі артефакти адаптації.
- Поєднувати стабільну базову модель зі спеціалізацією для tenant або продукту.

## Компроміси й обмеження

Adapters залишаються пов’язаними з архітектурою та ревізією базової моделі. Підтримка завантаження, об’єднання, stacking або перемикання adapters у runtime різниться. Поєднання кількох adapters може створювати взаємні перешкоди, а не додавати можливості.

## Типові помилки

- Сприйняття adapters як незалежних від базової моделі.
- Завантаження несумісних назв або розмірностей модулів.
- Stacking adapters без оцінювання взаємодій.
- Публікація adapter без документування потрібних базових ваг і ліцензії.
