# Pruning

Pruning вилучає ваги, нейрони, attention heads, шари чи інші структури моделі, визнані непотрібними для цільової мети.

## Переклади

- [English](../../)
- Українська

## Основна ідея

Неструктурований pruning установлює окремі ваги в нуль, тоді як структурований вилучає більші одиниці, які обладнання й runtimes можуть ефективніше використати. Після pruning моделі часто додатково fine-tune для відновлення якості.

## Практичне застосування

- Зменшити розмір моделі або обсяг обчислень.
- Дослідити внесок компонентів у задачу.
- Створити менші архітектури, зручні для обладнання.
- Поєднати з квантуванням або дистиляцією.

## Компроміси й обмеження

Розріджені ваги не гарантують прискорення, якщо runtime й обладнання не підтримують відповідний шаблон sparsity. Агресивний pruning може пошкодити загальні можливості або створити крихкість у спеціалізованих задачах. Повторне навчання й оцінювання можуть коштувати більше, ніж вибір наявної меншої моделі.

## Типові помилки

- Публікація кількості нульових ваг як практичного прискорення.
- Pruning без вимірювання якості на різноманітних задачах.
- Припущення, що кожен шар робить однаковий внесок.
- Ігнорування підтримки структурованої чи неструктурованої sparsity у runtime.
