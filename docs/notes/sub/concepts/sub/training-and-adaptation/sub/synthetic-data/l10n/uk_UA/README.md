# Синтетичні дані

Синтетичні дані — дані для навчання або оцінювання, згенеровані правилами, симуляціями, процедурними системами чи іншими моделями, а не зібрані безпосередньо з реальних подій.

## Переклади

- [English](../../)
- Українська

## Основна ідея

Синтетична генерація може розширити покриття, створити рідкісні edge cases або уникнути розкриття реальних персональних даних. Вона найкорисніша, коли правила генерації контрольовані, а outputs перевіряються незалежними критеріями.

## Практичне застосування

- Створювати кандидатні пари «інструкція — відповідь» для review.
- Симулювати помилки або рідкісні експлуатаційні сценарії.
- Балансувати класи, недостатньо представлені в реальних даних.
- Створювати прототипи зі збереженням приватності.
- Генерувати test cases із відомою очікуваною структурою.

## Компроміси й обмеження

Синтетичні дані успадковують помилки й упередження генератора. Повторювані згенеровані моделлю шаблони можуть звужувати різноманітність або навчати модель імітувати артефакти замість реального домену. Синтетичні дані не є автоматично анонімними, якщо відтворюють запам’ятовану інформацію.

## Типові помилки

- Навчання безпосередньо на неперевіреному output моделі.
- Використання тієї самої моделі для генерації та оцінювання всіх прикладів.
- Повна заміна реальних валідаційних даних синтетичними.
- Вимірювання розміру dataset замість покриття й правильності.
