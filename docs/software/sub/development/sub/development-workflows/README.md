# Development Workflows

Tools and frameworks that structure AI-assisted software delivery from product intent and requirements through planning, implementation, validation, and maintenance.

## Scope

Use this node for software whose primary role is defining or enforcing a development lifecycle through durable artifacts, explicit stages, and repeatable quality gates.

A tool belongs here when it organizes how development work moves between requirements, specifications, plans, tasks, implementation, review, and iteration. Integration with coding agents does not make the tool itself an agent.

Use other software nodes when the primary role is different:

- `code-editors/` for editor products and editor-specific extensions;
- `agents/` for standalone agent-like systems;
- `agent-orchestration/` for runtimes and control planes that coordinate agents;
- `workflow-engines/` for general AI workflow construction and execution.

## Tools

- [`spec-kit/`](./sub/spec-kit/) — GitHub's open-source Spec-Driven Development toolkit for turning product intent into specifications, plans, tasks, validation steps, and coding-agent implementation workflows.
