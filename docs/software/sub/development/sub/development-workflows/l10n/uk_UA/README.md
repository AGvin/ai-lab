# Робочі процеси розробки

Інструменти та фреймворки, які структурують розробку програмного забезпечення за допомогою ШІ — від задуму продукту й вимог до планування, реалізації, перевірки та супроводу.

## Переклади

- [English](../../)
- Українська

## Область застосування

Використовуйте цей вузол для програмного забезпечення, основна роль якого полягає у визначенні або забезпеченні життєвого циклу розробки через довговічні артефакти, чітко визначені етапи та повторювані контрольні точки якості.

Інструмент належить сюди, якщо він організовує рух роботи з розробки між вимогами, специфікаціями, планами, завданнями, реалізацією, перевіркою та ітераціями. Інтеграція з агентами програмування не робить сам інструмент агентом.

Якщо основна роль відрізняється, використовуйте інші вузли програмного забезпечення:

- [`code-editors/`](../../../code-editors/l10n/uk_UA/) — для редакторів і розширень, специфічних для редакторів;
- [`agents/`](../../../../../agents/l10n/uk_UA/) — для автономних агентоподібних систем;
- [`agent-orchestration/`](../../../../../agent-orchestration/l10n/uk_UA/) — для середовищ виконання та площин керування, які координують агентів;
- [`workflow-engines/`](../../../../../workflow-engines/l10n/uk_UA/) — для загального конструювання й виконання робочих процесів ШІ.

## Інструменти

- [`spec-kit/`](../../sub/spec-kit/l10n/uk_UA/) — набір інструментів GitHub з відкритим кодом для розробки на основі специфікацій, який перетворює задум продукту на специфікації, плани, завдання, кроки перевірки та робочі процеси реалізації агентами програмування.
