# Factory Droid

Агент розробки програмного забезпечення Factory AI для командних інженерних процесів.

- [English](../../)
- Українська

```text

Основний сценарій використання: підтримувати розробку ПЗ через агентне програмування, review, автоматизацію та інтеграції командних процесів
```

## Огляд

Factory Droid є частиною агентної платформи Factory для розробки програмного забезпечення.

Він орієнтований на командні інженерні процеси, а не на автономну функцію доповнення коду. Розглядайте його як продукт агента програмної інженерії, що може брати участь у розробці, review й автоматизації в репозиторіях і командних інструментах.

## Відповідність AI Lab

Factory Droid належить до `agents/`, оскільки його основна задокументована роль — агентоподібна система розробки ПЗ.

Використовуйте його як орієнтир для:

- пропрієтарних платформ агентів програмної інженерії;
- командно орієнтованих агентних процесів розробки;
- автоматизації репозиторіїв і review;
- enterprise governance агентної розробки ПЗ;
- порівняння з Devin, coding agent GitHub Copilot, Claude Code і OpenAI Codex.

## Примітки щодо оцінювання

Перед впровадженням оцініть:

- модель доступу через обліковий запис, команду та enterprise;
- області дозволів репозиторію та організації;
- межі review, commit, pull request й автоматизації;
- інтеграції з наявними інженерними системами;
- можливість аудиту створених змін;
- розкриття облікових даних і секретів;
- контрольні точки схвалення людиною для merge, deploy і release.
